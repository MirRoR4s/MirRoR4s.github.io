deserialize()

到这里的话，Shiro 针对 cookie 中 rememberMe 字段值如何操作我们已经很清楚了：

  1. 对 rememberMe 进行 base64 解码，如果解码结果不为 deleteMe 进入第二步
  2. 调用 AES 算法对 rememberMe 进行解密，加密的密钥硬编码在源码中。
  3. 将解密结果反序列化返回。

漏洞利用

思路很简单，因为我们知道这个解密过程会有一个反序列化嘛，所以可以利用之前学习过的一些 CC链、URLDNS 啥的进行反序列化攻击。

基本操作就是将 payload 进行 AES 加密然后 base64 编码然后放入到 rememberMe 字段中即可。

那么 exp 的编写就涉及到三个问题：

  1. AES 加密
  2. base64 编码
  3. 本身的 payload 编写

前两个问题可以利用 python 的第三方库轻松解决，最后一个需要借助前面学习的知识。

  • exp 的编写的话其实芜风师傅有个点没有说到的，就是加解密过程中采用的工作模式是什么，Shiro550 默认是 CBC 模式。
  • CBC 模式需要一个初始向量，rememberMe base64编码前的前十六个字节用作 iv，这一点我还没有看源码。
