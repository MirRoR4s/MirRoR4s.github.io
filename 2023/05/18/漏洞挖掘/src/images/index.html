src 文章分析

支付逻辑漏洞

实战某src支付漏洞0元购

  • 某 src 支付漏洞 0 元购

这个漏洞基本操作就是下单商品然后抓包得到下单操作对应的 HTTP 请求，在请求中含有商品价格、账号余额等参数。但是我们可以修改余额对应的参数值，使得购买的时候用余额支付。

某src支付逻辑有误导致任意支付

  • https://mp.weixin.qq.com/s?__biz=Mzg5MDk3NDE2Mw==&mid=2247483686&idx=1&sn=c064ff86ca6da19ae90508571625594a&chksm=cfd53b9af8a2b28caa6072c19e485fc4fef099809d752cabb2e11ff8f4db9ed1a9f90d724123&cur_album_id=2912733224823652356&scene=189#wechat_redirect

这个漏洞比较有意思，依旧是下单页面抓包进行操作，但是此时修改所有的参数都会发现有校验，但是关键的点来了：

挖掘者查看了返回包的信息并分析出了其中蕴含着的关键字段，然后它复制了返回包的所有参数到请求包中，同时修改关键字段的值，最后也是成功完成了任意支付。

这件事情告诉我们返回包的审查也是十分重要的！！
