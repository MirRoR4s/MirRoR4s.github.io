Git 是什么？

  • git 就是一个版本控制系统，用于追踪和管理代码文件的变更。

感觉 git 就解决三个问题：

  1. 文件的同步
  2. 文件的协作
  3. 历史版本

首先阐述的是如何解决历史版本的问题，这里主要的应用场景是我们修改了文件的某个部分，比如删除了某几行。但是在之后忽然又发现我们需要恢复这几行，这个时候该如何回退到之前的老版本的问题。git 采用的策略也很简单，就是弄一个日志来记录文件的所有变更，想要回退老版本，根据日志进行操作就行了。

之后说到的是文件的同步问题。应用场景是小蓝变更了文件，小红如何拿到这个最新的文件的问题。git 采用的做法是搭建一个 git 服务器，并以此作为小红、小蓝之间的桥梁。具体地说，小蓝变更完文件后，需要将这个文件提交到 git 服务器上，然后小红可以从 git 服务器上再获取这个文件。这样我们就做到了文件的同步。值得注意的是如果小红、小蓝变更的是同一个文件，那么小红此时不可以直接向服务器提交他变更过的文件，他必须先从服务器拿到小蓝变更的文件。
