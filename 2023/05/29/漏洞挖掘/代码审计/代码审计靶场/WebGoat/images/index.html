配置 Dockerfile

修改 Dockerfile，在 ENTRYPOINT 命令中增加调试相关参数，如下所示：

其中 -agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=*:5005 是启用远程调试的参数。这将在容器内开放 5005 端口，作为调试端口。

在 Intel IDEA 中，您需要创建一个新的远程配置，并使用容器的 IP 地址和 5005 端口来进行连接和调试。

运行容器时，请确保将 -p 5005:5005 参数添加到 docker run 命令中以映射容器的端口。

配置 IDEA

  1. 获取 WebGoat 源代码

您可以通过以下命令从 WebGoat GitHub 存储库中克隆源代码：

  1. 将源代码导入 Intel IDEA

在 Intel IDEA 中，选择 File -> Open，然后选择从第一步中克隆的 WebGoat 目录，并等待 Intel IDEA 将项目导入。

  1. 在 IDEA 左侧的 Run 面板中，点击 Edit Configurations...。
  2. 在弹出的窗口中，点击左上角的 + 按钮，选择 Remote。
  3. 在创建一个新的 Remote 配置后，设置 Name 为您喜欢的名称，Debugger mode 选择 Attach to remote JVM，Host 填写 localhost（如果是在远程服务器上运行，则需要填写服务器的 IP 地址），Port 填写您在 docker run 命令中设置的调试端口，即 5005。
  4. class path 就选择刚打开的 WebGoat 目录
  5. 点击 OK 按钮保存并退出配置窗口。
  6. 启动 WebGoat 容器并等待其启动完毕。
  7. 在 IDEA 左侧的 Run 面板中，选择您刚刚创建的 Remote 配置。然后，单击绿色的 Debug 按钮启动远程调试。
  8. 如果一切正常，您应该能够连接到 WebGoat 的远程调试进程。现在，您可以开始像本地调试一样操作，并在 IDEA 中进行断点调试和查看变量等。
