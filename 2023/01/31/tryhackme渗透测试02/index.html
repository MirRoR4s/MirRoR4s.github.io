渗透测试 Content Discovery 篇章。

一. 前言

这一篇章主要是学会如何发现 web 服务器上隐藏的和私有的可能导致漏洞的内容/信息，共计 12 个挑战。

二. 挑战

2.1 什么是内容发现

在讨论什么是内容发现之前，先说说在一个 web 应用程序的上下文环境中，什么是内容。

内容可以是很多东西，比如说文件、视频、图片、备份文件、网站特征。内容发现就是要找到可能导致漏洞的信息。

当我们在讨论内容发现的时候，我们不是在说我们能在网站上看到的明显的信息，而是那些没有直接展示给我们、预期不对公众展示的信息。

如何进行内容发现呢？内容发现有三种常用的方法，它们分别是：

  1. 手工
  2. 自动
  3. OSINT 开源情报

2.2 手动发现之 robots.txt

robots.txt文件是一个文档，它告诉搜索引擎哪些页面可以显示在搜索引擎结果中，哪些页面不可以显示在搜索引擎结果中，或者禁止特定的搜索引擎完全抓取网站。通常的做法是限制某些网站区域，这样它们就不会显示在搜索引擎结果中。这些页面可能是针对网站客户的管理门户或文件等区域。这个文件给了我们一个网站上的位置列表，所有者不希望我们作为渗透测试人员发现这些位置。

2.3 手动发现之 Favicon

favicon是显示在浏览器地址栏或选项卡上的一个小图标，用于标记网站。

有时，当使用框架构建一个网站时，作为安装的一部分的图标会被遗留下来，如果网站开发人员没有用自定义的图标替换它，那这可以让我们知道使用的是什么框架。OWASP 拥有一个通用框架图标的数据库，您可以使用它来检查目标图标。一旦我们了解了框架堆栈，我们就可以使用外部资源来发现更多关于它的信息(见下一节)。

实操

2.4 手动发现之 Sitemap.xml

不像robots.txt文件，它限制了搜索引擎爬虫可以查看的内容，sitemap.xml文件给出了网站所有者希望在搜索引擎上列出的每个文件的列表。这些页面有时会包含网站中一些比较难导航的区域，甚至会列出一些当前网站不再使用但仍在幕后工作的旧网页。

2.7 OSINT Google Hacking/Dorking

还有一些外部资源可以帮助你发现目标网站的信息;这些资源通常被称为OSINT或(开源情报)，因为它们是免费的收集信息的工具:

Google Hacking / Dorking

谷歌 hacking / Dorking 利用谷歌的高级搜索引擎功能，允许您挑选自定义内容。举个例子，通过使用 site: 过滤器，可以挑选出来自特定域名的搜索结果。（比如 site: tryhackme.com）还可以在 site: xxx 过滤器后附加上一些关键字，这用来表明返回的搜索结果中要包含这个关键字。比如 ste:tryhackme.com admin 意思是返回来自于 tryhackme.com 这个域名并且内容中包含了 admin 关键字的网页。

2.8 OSINT Wappalyzer

Wappalyzer (https://www.wappalyzer.com/) 是一个在线工具和浏览器扩展，可以帮助识别网站使用的技术，如框架、内容管理系统(CMS)、支付处理器等等，甚至还可以找到版本号。

2.9 OSINT Wayback Machine

Wayback Machine (https://archive.org/web/) 是一个网站的历史档案，可以追溯到90年代末。你可以搜索一个域名，它会显示该服务抓取网页并保存内容的所有时间。这项服务可以帮助发现可能仍然活跃在当前网站的旧页面。

2.10 OSINT GitHub

要理解 GitHub ，你首先需要理解 Git 。Git 是一个版本控制系统，用于跟踪项目中文件的更改。这使得团队工作更容易，因为您可以看到每个团队成员正在编辑什么以及他们对文件做了哪些更改。当用户完成更改时，他们将使用消息提交这些更改，然后将它们推回到中心位置(存储库)，以便其他用户将这些更改拉到本地计算机。GitHub 是 Git 在互联网上的托管版本。存储库可以设置为公共或私有，并具有各种访问控制。

您可以使用 GitHub 的搜索功能来查找公司名称或网站名称，以尝试定位属于目标的存储库。一旦被发现，你就可以访问源代码、密码或其他你还没有发现的内容。

2.11 OSINT S3 Buckets

S3 bucket是亚马逊AWS提供的一种存储服务，允许人们将文件甚至静态网站内容保存在通过HTTP和HTTPS访问的云中。文件的所有者可以设置访问权限，使文件公开、私有甚至可写。有时这些访问权限设置不正确，无意中允许访问不应该向公众提供的文件。

S3 bucket 的格式是 http(s)://{name}.s3.amazonaws.com，其中 {name} 由所有者决定，例如 tryhackme-assets.s3.amazonaws.com。S3存储桶可以通过多种方式被发现，比如在网站的页面源、GitHub 存储库中找到 url，甚至自动化这个过程。一种常用的自动化方法是使用公司名称后面跟着常用的术语，如 {name}-assets、{name}-www、{name}-public、{name}-private 等。

2.12 自动发现

2.12.1 什么是自动发现

自动发现是使用工具而不是手动发现内容的过程。这个过程是自动化的，因为它通常包含数百、数千甚至数百万个到 web 服务器的请求。这些请求检查一个文件或目录是否存在于一个网站上，让我们访问我们以前不知道存在的资源。这个过程可以通过使用一个叫做单词列表（wordlists）的资源来实现。

2.12.2 单词列表是什么？

单词列表只是包含一长串常用单词的文本文件;它们可以涵盖许多不同的用例。例如，密码单词列表将包含最常用的密码，而我们在本例中查找内容，因此需要一个包含最常用目录和文件名的列表。THM AttackBox上预装的一个很好的单词列表资源是 Daniel Miessler 管理的 https://github.com/danielmiessler/SecLists。
