Intro to Offensive Security

防御性安全前言 Intro to Defensive Security

本篇介绍了一些 defensive security 和 相关的主题，比如 威胁情报、SOC、DFIR、SIEM 等。

防御安全简介 Introduction to Defensive Security

Offensive security（进攻性安全？不知道咋翻译这个名词）聚焦于一件事情：侵入系统。侵入系统可以通过利用漏洞、滥用不安全的设置、利用未强制的访问控制策略等方式实现。红队和渗透测试人员专门负责进攻性安全。

Defensive security（防御性安全）在某种程度上与进攻性安全相反，因为其涉及到两个主要的任务：

  1. 阻止入侵的发生
  2. 在入侵发生时检测入侵并正确响应

蓝队是 defensive security 情形的一部分

一些和 defensive security 相关联的任务包括：

  • 用户网络安全意识：对用户进行网络安全培训有助于防止针对其系统的各种攻击
  • 记录和管理资产：我们需要知道我们必须正确管理和保护的系统和设备的类型
  • 更修和修补系统：确保计算机、服务器和网络设备正确地更新以及修补任何已知的漏洞或是弱点
  • 设置预防性安全设备：防火墙和入侵防御系统（IPS）是预防性安全的重要组成部分，防火墙控制哪些网络流量可以进入系统或网络，哪些可以离开系统或网络。IPS会阻断任何匹配当前规则和攻击特征的网络流量。
  • 设置日志和监控设备：如果没有对网络进行适当的日志记录和监控，就不可能检测到恶意活动和入侵。如果一个新的未经授权的设备出现在我们的网络上，我们应该能够知道。

关于 defensive security 防御安全还有很多，上面的列表只涵盖了一些常见的主题。

In this room, we cover:

  • Security Operations Center (SOC) 安全运营中心
  • Threat Intelligence 威胁情报
  • Digital Forensics and Incident Response (DFIR) 电子取证和应急响应
  • Malware Analysis 恶意软件分析

入侵防御系统(IPS)是一种能够主动检测和阻止入侵行为的设备或应用。它们通常部署在受保护资产的前面，阻止任何潜在的威胁到达目标。

安全运营中心(SOC)是一个由IT安全专业人员组成的团队，负责监控、预防、检测、调查和响应公司网络和系统中的威胁。

Areas of Defensive Security

在这项任务中，我们将讨论与防御安全相关的两个主要主题：

  • 安全行动中心(SOC)，在那里我们覆盖威胁情报
  • 电子取证和事件响应(DFIR)，其中我们还包括恶意软件分析

安全运营中心（SOC）

SOC (Security Operations Center)是一个由网络安全专业人员组成的团队，负责监控网络及其系统，以检测恶意网络安全事件。SOC 的一些主要兴趣领域是:

  • 漏洞：无论何时发现系统漏洞(弱点)，都必须通过安装适当的更新或补丁来修复它。当修复程序不可用时，应该采取必要的措施来防止攻击者利用它。虽然修复漏洞对SOC至关重要，但并不一定要分配给他们（也就是 SOC 并不一定还要负责修复漏洞）

  • Policy violations：我们可以把安全策略看作是保护网络和系统所需的一组规则。例如，如果用户开始将机密公司数据上传到在线存储服务，这可能是违反策略的。

  • 未经授权的活动:考虑这样一种情况，用户的登录名和密码被盗，攻击者使用它们登录到网络。SOC需要检测到这样的事件，并在进一步破坏之前尽快阻止它

  • 网络入侵:无论你的安全性有多好，总会有被入侵的机会。当用户单击恶意链接或攻击者利用公共服务器时，就可能发生入侵。无论哪种方式，当入侵发生时，我们必须尽快检测到它，以防止进一步的破坏。

Security operations 安全操作涵盖各种任务以确保安全;其中一项任务就是威胁情报。

Threat Intelligence 威胁情报

在当前语境下，情报是指你收集到的明面上的和潜在的敌人的信息，威胁是可以破坏或对系统产生不利影响的任何操作。威胁情报旨在收集信息，帮助公司更好地应对潜在的敌手，其目的是实现威胁知情防御。

不同的公司有不同的敌手，一些攻击者可能试图从移动运营商窃取客户数据。然而，其他敌手对停止一家炼油厂的生产感兴趣（这里指的是攻击者企图让公司丧失提供服务的能力）。例如常见的敌手可能包括出于政治原因的民族国家网络军队和出于金融目的的勒索软件组织。Based on the company (target), we can expect adversaries.

情报需要数据，所以我们必须收集、处理和分析数据。我们可以从本地来源（比如网络日志）或是公有来源（比如论坛）收集数据。数据处理的目的是将它们排列成适合分析的格式。分析阶段试图找到更多关于攻击者及其动机的信息，此外，它的目的是创建一个建议和可操作步骤的列表。

了解你的对手可以让你了解他们的战术、技术和程序。作为威胁情报的结果，我们可以识别威胁行为者(敌手)，预测他们的活动，因此，我们将能够减轻他们的攻击并准备响应策略

这一小节是关于电子取证和应急响应的（DFIR），我们将会涵盖以下内容：

  • 电子取证
  • 应急响应

电子取证

取证是调查犯罪和查明真相的科学应用，随着信息系统的使用和传播，例如计算机和智能手机，取证诞生了一个调查相关犯罪的新分支：计算机取证——后来演变成了电子取证。

在防御性安全 defensive security 中，电子取证的重点转移到分析攻击及其肇事者的证据以及其他领域，如知识产权盗窃、网络间谍活动和拥有未经授权的内容。因此，电子取证将专注于不同的领域，如:

  • 文件系统：分析系统存储的数字取证映像(低级副本)可以揭示许多信息，例如已安装的程序、已创建的文件、已部分覆盖的文件和已删除的文件。
