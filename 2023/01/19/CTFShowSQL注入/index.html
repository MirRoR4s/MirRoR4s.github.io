前言

以下题目大都过滤了空格，一般代替空格的字符串有

  • %09
  • %0a
  • %0b
  • %0d
  • %0c

web176

这里其实是对 select 进行了过滤，大小写就能绕过。但是也可以如下拿 flag

web177

这里应该是过滤掉了空格，只要存在空格就把整个语句都清空。用 %09 替代空格即可

踩坑点：

使用 –+ 作为注释符会失效，因为最后的 + 在url解码后被识别为空格，然后触发了题目的 waf

所以可以将 # 编码为 %23 来作为注释符，不编码的话 # 在url中有特殊含义会有些问题

web178

依旧过滤了空格，上一题的答案还能用。不知道为什么这样的语句行不通，所以具体过滤了什么呢？

web179

依旧是过滤了空格，但是现在 %09 不好用啦。但是可以用 %0c 代替空格

web180

这一关过滤了注释符 %23,而且空格还是只能用 %0c 代替。那我们只能使用另一个注释符–空格了，将空格替换成 %0c 就好
