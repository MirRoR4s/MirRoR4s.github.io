Knight Search#

随便看了下发现 post /home 的时候会弹出一个 keyerror，说是缺少 filename 这个key。题目描述已经说了这是一个文件搜索题，所以猜测应该是传入文件名然后回显文件内容，并且测试 post 了 filename 确实不报错了，尽管没有内容。

但是在报错信息中找到了当前的 python 文件名 app.py，post 这个文件名成功回显了文件的内容。

绕过 ../ 只需要两次 url 编码即可，但是 current_app.open_resource(filename) 限制了只能读取 current_app 目录下的文件，在这里想了很久被卡住了。期间想到过在报错信息的控制台，但是不知道咋搞 PIN 码放弃了，没想到赛后题解真的是 PIN 码！！下面就是生成 PIN 码的脚本

误会了，open_resource 并没有限制只能读取 current_app 目录下的文件，两次 url 编码可以绕过，但是需要在 python 下进行 post 发包，千万别在浏览器或者 burp！！！我也不知道为什么~

只能读文件还是弄不到 flag 的，所以需要生成 PIN 码进行命令执行

关于 PIN 码的分析和生成可以查看这篇：

https://xz.aliyun.com/t/8092#toc-3

生成完 PIN 码之后就去登录到控制台然后执行系统命令获得 flag

值得注意的是 system 命令拿不到命令执行的结果，还是要借助 popen

I Love Pi#

题目描述#

Isaac Newton left me this piece of code and a message. Can you help me decode this…

Demo Flag : KCTF{FL4G_H3R3}

Attachments: Link

Author: sweet_dreams

题目内容#

题目分析#

可以看到题目每次从一个长度为10的列表中取出某个数a，然后从flag中取出长度为a的字串，并进行a次编码。解码很容易，但是题目没有将这个列表告诉我们，这就使得我们不知道每一轮被编码的字串的长度。
