题目漏洞分析

题目会接收一个名字为 id 的参数，然后进行SQL查询，注意到这里是有一个黑名单的过滤的。

同时查询完毕之后还有一个 in_array() 函数过滤，其会判断传入经过 stop_hac k函数过滤后的 id 是否在白名单内，如果在就把 sql 语句的查询结果告诉我们，否则就 die。

in_array() 函数在这里第三个参数并没有设置 true，所以比较是一个弱类型的比较，那么我们传入 1a 和 1 比较会被认为是相等的，所以这一步很容易就可以绕过。

关键的问题在于如何构造sql语句，观察stop_hack 会发现 union 被过滤了，那么联合注入就不行。同时 or 被过滤，导致不能查 information_schema 表。
