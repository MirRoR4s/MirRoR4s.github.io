ezphp#

题目描述#

小楠楠学习了php8，来看看php高版本有啥特性吧

题目内容#

<?php
highlight_file(__FILE__);
$num = $_GET['num'];
if(is_string($num) && strlen($num) < 5 && strpos($num,'111') === false && strpos($num,'0') === false && eval("return 111===${num};")){
    readfile('/flag');
}

?>

题目分析#

源码很简单，满足如下几个条件就会读取 flag

  • num 是字符串
  • num 的长度小于 5
  • num 中不包含 111
  • num 中不包含 0
  • eval执行 111 强等于 num 为 真

解题步骤#

url 传参如下

?num = 1||1

明显前几个条件都是满足的，那为什么最后一个条件是真呢？

首先在 php 中 || 表示 or 也就是逻辑或的意思。将 num 的值带入 eval 语句中得到：

eval(return 111===1||1;)//由于 === 的优先级高于 || ，php会选进行判断111===1是否成立，然后再和1做计算

111 强等于 1 不为真，但是 or 1 之后就为真了，因为 1 为真！如果我们传入 1||0 那么结果就是 false：

题目分析#

查看 Dockerfile 发现

  • 基础镜像是 mcr.microsoft.com/dotnet/aspnet:6.0
  • cat 命令加了 suid 权限，这表示的含义可以查阅知乎：linux文件用户权限中的SUID有什么用？

这意味着 如果 /bin/cat 程序的所有者是 root 的话，那么给这个程序增加了 suid 权限就使得普通用户在下执行这个程序时可以查看一些需要 root 权限才能查看的文件。

关于 chmod 命令各种选项含义可以看这篇文章：Linux权限详解

紧接着浏览 start.sh 发现：

  • /flag 文件的权限是 700，这意味着只有该文件的所有者具有读写执行权限。
  • 服务启动调用的是 WebApplication1.dll 文件

所以接下来我们反编译一下这个 dll 文件查看一下源代码。题目描述中说了这是用 c#写的web程序，也就是初始程序是 .NET 平台的程序，所以我们可以利用 dnSpy 来进行反编译。

打开 dnSpy.exe 导入 webApplication.dll
