本次针对 xxe 进行一个代码审计，打开 007-xxe.jsp 分析源码。

007-xxe.jsp

漏洞证明

request.getRequestURL() 是获取到请求的 url 链接，我们这里是 http://127.0.0.1:8080/vulns/007-xxe.jsp

之后拼接上 windows_querystring 这个字符串，该字符串的定义就在源码的开头。

这是我们 xxe 的 payload，将其解码看看：

?data=<?xml version="1.0" encoding="ISO-8859-1"?><!DOCTYPE foo [   <!ELEMENT foo ANY >  <!ENTITY xxe SYSTEM "file:///c:/windows/win.ini" >]><foo>&xxe;</foo>

总结一下代码所做的事情：

  1. 实例化一个 DocumentBuilderFactory 对象 doFactory

  2. 调用 doFactory 对象的 newDocumentBuilder 方法得到一个 DocumentBuilder 类的实例对象 docBuilder

  3. 调用 docBuilder 的 parse 方法对用户传入的数据进行解析，最后会返回一个 Document 类的实例对象 doc，也就是一个 DOM。

Java 中的 Document 是一个接口，代表整个 HTML 或 XML 文档。
