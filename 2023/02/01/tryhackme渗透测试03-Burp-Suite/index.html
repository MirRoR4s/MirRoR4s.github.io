2.2 Burp Suite 是什么？

简单地说:Burp Suite 是一个用 Java 编写的框架，旨在为 web 应用程序渗透测试提供一站式服务。在许多方面，这一目标都是实现的，因为 Burp 在很大程度上是一种用于实际应用程序安全评估的行业标准工具。

Burp Suite 在评估移动应用程序时也非常常用，因为它对 web 应用程序测试非常有吸引力的相同功能几乎完美地转化为对大多数移动应用程序的 API（应用程序编程接口）的测试。

在最简单的层面上，Burp 可以捕获和操纵攻击者和 Web 服务器之间的所有流量：这是框架的核心。在捕获请求后，我们可以选择将它们发送到 Burp Suite 框架的其他各个部分—— 我们将在接下来的会议室中介绍其中的一些工具。这种在将 web 请求发送到目标服务器之前拦截、查看和修改 web 请求的能力（或者在某些情况下，在浏览器接收到响应之前），使 Burp Suite 非常适合任何类型的手动 web 应用测试。

有各种不同版本的 Burp Suite 可供选择。我们将使用 Burp Suite 社区版，因为它可以免费用于任何（合法）非商业用途。Burp Suite Professional 版和 Enterprise 版都需要昂贵的许可证，但具有强大的额外功能：

  • 自动漏洞扫描器
  • 不受速率限制的 fuzzer/ bruteforcer
  • 保存项目以备将来使用；报告生成
  • 允许与其他工具集成的内置 API
  • 无限制访问以添加新扩展以实现更强的功能
  • 访问 Burp Suite Collaborator（有效地提供了一个独立的请求捕获器或在 Portswigger 拥有的服务器上运行）

简而言之，Burp Pro 是一个非常强大的工具，这就是为什么它的一年订阅费为每位用户 319 英镑/ 399 美元。因此，Burp Pro 通常只供专业人士使用(许可证通常由雇主提供)。

Burp Suite Enterprise 略有不同。与社区版和专业版不同，Burp Enterprise 用于连续扫描。它提供了一个自动扫描仪，可以定期扫描 Web 应用程序中的漏洞，与 Nessus 等软件执行自动基础设施扫描的方式大致相同。与允许您从自己的计算机执行手动攻击的其他版本的 Burp Suite 不同，Enterprise位于服务器上，并不断扫描目标 web 应用程序以查找漏洞。

由于这两个版本的 Burp Suite 都涉及高昂的成本，我们将坚持 Burp Suite 社区提供的核心功能集。

Burp Suite for Windows 是许多演示的截图；但是，这与安装在AttackBox 上的 Burp Suite 副本没有区别。

2.3 Burp 社区版的特性

虽然与专业版相比，Burp 社区的功能集相对有限，但它仍然有许多优秀的工具可用。其中包括：

  • Proxy：Burp Suite 最著名的方面，Burp 代理允许我们在与 web 应用程序交互时拦截和修改请求/响应。

  • Repeater：第二个最著名的 Burp 特性—— Repeater ——允许我们多次捕获、修改和重新发送相同的请求。这一特性绝对是非常宝贵的，尤其是当我们需要通过试错（例如在 SQLi ——结构化查询语言注入中）或测试端点的功能以查找缺陷时。

  • Intruder：虽然在Burp社区中受到严格的速率限制，但入侵者允许我们向端点发送请求。这通常用于暴力攻击或模糊端点。

  • Decoder：虽然比前面提到的功能使用更少，但解码器在转换数据时仍然提供了一项有价值的服务——无论是解码捕获的信息，还是在将有效载荷发送到目标之前对其进行编码。虽然有其他服务可以完成相同的工作，但直接在Burp Suite 内完成这项工作可能非常有效。

  • Comparer：顾名思义，比较器允许我们在单词或字节级别比较两段数据。同样，这并不是 Burp Suite 独有的功能，但通过一个键盘快捷键将（可能非常大）数据直接发送到比较工具中可以大大加快速度。

  • Sequencer：我们通常在评估令牌的随机性时使用 Sequencer，例如会话 cookie 值或其他所谓的随机生成数据。如果算法没有生成安全的随机值，那么这可能会为攻击开辟一些毁灭性的途径。

除了大量内置功能外，Java 代码库还使编写扩展以添加到 Burp 框架的功能变得非常容易。这些可以用 Java、Python（使用 Jython 解释器）或Ruby（使用Java Ruby解释器）编写。Burp Suite Extender 模块可以快速轻松地将扩展加载到框架中，并提供下载第三方模块的市场（称为“BApp Store”）。虽然其中许多扩展需要专业许可才能下载和加载，但仍有相当数量的扩展可以与 Burp 社区集成。例如，我们可能希望通过 Logger++ 模块扩展 Burp Suite 的内置日志功能。

2.7 Options 选项

在我们开始学习 Burp 代理之前，让我们看看可用于配置 Burp Suite 的选项。

  • 全局设置可以在顶部菜单栏的用户选项（User Options）选项卡中找到。

每次打开 Burp Suite 时，“用户选项”选项卡中提供的选项都将适用。相反，“项目”选项仅适用于当前项目。鉴于我们无法在 Burp 社区版中保存项目，这意味着我们的项目选项将在每次关闭 Burp 时重置。

有太多的选项，无法全部涵盖，所以让我们只看可用的类别，并强调一些更重要的设置。我们将从 User Options 选项卡开始。

这里的设置适用于全局（即，它们控制 Burp Suite 应用程序，而不仅仅是项目）。也就是说，其中许多可以在项目设置中覆盖。

User options 选项卡有四个主要的子部分：

  • Connections 子选项卡中的选项允许我们控制 Burp 如何与目标建立连接。 例如，我们可以为 Burp Suite 设置一个代理来连接； 如果我们想通过网络枢轴使用 Burp Suite，这将非常有用。
  • TLS 子选项卡允许我们启用和禁用各种 TLS（传输层安全性）选项，并为我们提供上传客户端证书的位置（如果 Web 应用程序要求我们使用证书进行连接）。
  • 一组基本选项：Display 允许我们改变 Burp Suite 的外观。 这里的选项包括更改字体和比例，以及为框架设置主题（例如暗模式）以及配置各种选项以处理 Repeater 中的渲染引擎（稍后会详细介绍！）。
  • Misc 子选项卡包含各种设置，包括键绑定表（热键），它允许我们查看和更改 Burp Suite 使用的键盘快捷键。 建议您熟悉这些设置，因为使用快捷键可以大大加快您的工作流程。

使用这些选项，我们可以自定义 Burp 安装以满足我们的个人喜好。

  • HTTP 子选项卡定义了 Burp 如何处理 HTTP 协议的各个方面：例如，它是否遵循重定向或如何处理异常响应代码。

  • 与“User options”部分的等效选项卡相比，杂项 Misc 子选项卡中的选项更少。 这里的许多选项也只有在您有权访问 Burp Pro 时才可用（例如那些配置 Collaborator 的选项）。 也就是说，有几个与日志记录和嵌入式浏览器相关的选项（我们将在几个任务中查看）非常值得细读。

2.8 Burp Proxy

Burp 代理是 Burp Suite 中最基本（也是最重要的！）的工具。它使我们能够捕捉自己和目标之间的请求和响应。然后，在允许它们继续到达目的地之前，可以对它们进行操作或发送到其他工具进行进一步处理。

例如，如果我们通过 Burp 代理向 https://tryhackme.com 发出请求，我们的请求将被捕获，并且在我们明确允许之前，请求不会被发送到 TryHackMe 服务器。 我们可以选择对服务器的响应做同样的事情，尽管默认情况下这不是活动的。 这种拦截请求的能力最终意味着我们可以完全控制我们的网络流量——这是测试网络应用程序时非常宝贵的能力。

在使用代理之前，我们需要进行一些配置，但让我们从查看界面开始。

2.9 通过 Proxy 代理 进行连接（FoxyProxy）

你已经看到了这个理论； 现在是时候开始为自己使用代理了。

有两种方法可以通过 Burp Suite 代理我们的流量。

  1. 我们可以使用嵌入式浏览器（我们将在稍后的任务中介绍）。
  2. 我们可以配置我们的本地网络浏览器来通过 Burp 代理我们的流量； 这种情况更为常见，因此将成为本任务的重点。

Burp 代理通过在 127.0.0.1:8080（默认）上打开一个 Web 界面来工作。 正如这是一个“代理”这一事实所暗示的，在我们开始使用 Burp 拦截它之前，我们需要通过该端口重定向所有浏览器流量。 我们可以通过更改浏览器设置或更常见的方法是使用名为 FoxyProxy 的 Firefox 浏览器扩展来做到这一点。 FoxyProxy 允许我们保存代理配置文件，这意味着我们可以通过单击快速轻松地切换到我们的 “Burp Suite” 配置文件，然后同样轻松地禁用代理。

注意：所有说明都将针对 Firefox 给出，因为这是 Kali Linux 和 TryHackMe AttackBox 的默认浏览器。 如果您在本地使用其他浏览器，那么建议您使用 AttackBox，否则您可能需要寻找替代方法来替代此任务中提供的方法。 如果您无法让代理在您的本地浏览器中工作并且不想使用 AttackBox，那么您可能希望跳到 Burp Suite 浏览器任务。

FoxyProxy 有两个版本：Basic 和 Standard。 这两个版本都允许您即时更改代理设置； 但是，FoxyProxy Standard 使您可以更好地控制通过代理发送的流量。 例如，它将允许您设置模式匹配规则以确定是否应代理请求：这比 FoxyProxy basic 提供的简单代理更复杂。

基础版足以满足我们的使用需求。 它已预安装并配置在 AttackBox 的 Firefox 浏览器中，因此如果您使用的是 AttackBox，请随时跳到此任务的最后一部分。

如果您使用自己的机器，可以在此处下载 FoxyProxy Basic。

请记住：当您连接到代理并打开代理拦截时，只要您发出请求，您的浏览器就会挂起。 当您学习使用 Burp Suite（事实上，后来！）时，一个非常常见的错误是不小心将拦截打开，因此无法通过浏览器发出任何 Web 请求。 如果您的浏览器挂起而您不知道原因：请检查您的代理！

2.12 Scoping and Targeting

最后，我们来到使用 Burp 代理最重要的部分之一：scope。

让 Burp 捕获我们所有的流量会变得非常乏味。 当它记录所有内容（包括我们未定位的网站的流量）时，它会混淆我们稍后可能希望发送给客户的日志。 简而言之，让 Burp 捕获所有内容会很快变成一个巨大的痛苦。

解决办法是什么？ 范围界定。

为项目设置范围允许我们定义代理和记录的内容。 我们可以限制 Burp Suite 只针对我们想要测试的 Web 应用程序。 最简单的方法是切换到“Target”选项卡，右键单击左侧列表中的目标，然后选择“Add To Scope”。 然后 Burp 会询问我们是否要停止记录任何不在范围内的内容——大多数时候我们想在这里选择“yes”。

我们现在可以通过切换到“Scope”子选项卡来检查我们的范围 scope（如上面的 GIF 所示）。

范围子选项卡允许我们通过包含或排除域/IP 来控制我们的目标。 这是一个非常强大的部分，因此花时间习惯使用它是非常值得的。

我们只是选择禁用范围外流量的日志记录，但代理仍会拦截所有内容。 要关闭此功能，我们需要进入“代理选项”子选项卡并从“拦截客户端请求”部分选择“并且 URL 在目标范围内”：

选择此选项后，代理将完全忽略任何不在范围内的内容，从而极大地清理通过 Burp 的流量。

2.13 Site Map and Issue Definitions

范围控制可能是 Target 选项卡最有用的方面，但它绝不是 Burp 这一部分的唯一用途。

Target 下有三个子选项卡：

  • 站点地图 Site map 允许我们在树结构中绘制出我们定位的应用程序。 我们访问的每个页面都会显示在这里，使我们能够通过浏览网络应用程序自动为目标生成站点地图。 Burp Pro 还允许我们自动抓取目标（即查看每个页面的链接，并使用它们绘制出尽可能多的网站——使用页面之间的链接可公开访问）； 但是，使用 Burp Community，我们仍然可以在执行初始枚举步骤时使用它来积累数据。如果我们想要映射 API，站点地图会特别有用，因为每当我们访问页面时，页面在加载时从中检索数据的任何 API 端点都会显示在这里。
  • 范围 scope ：我们已经看到范围子选项卡——它允许我们控制项目的 Burp 目标范围。
  • 问题定义 Issue Definitions ：虽然我们无法访问 Burp 社区中的 Burp Suite 漏洞扫描程序，但我们仍然可以访问它查找的所有漏洞的列表。 问题定义部分为我们提供了大量的 Web 漏洞列表（包括描述和参考），如果我们需要引用报告或帮助描述漏洞，我们可以从中提取。
