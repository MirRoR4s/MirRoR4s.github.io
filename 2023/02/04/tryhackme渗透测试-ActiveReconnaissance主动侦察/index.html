一. 前言

了解如何使用简单的工具（例如 traceroute、ping、telnet 和 Web 浏览器）来收集信息。

介绍

在网络安全模块的第一个房间，我们专注于被动侦察。在这个房间，我们专注于主动侦察和与之相关的基本工具。我们将学习使用网络浏览器来收集有关目标的更多信息。此外，我们还将讨论使用 ping、traceroute、telnet 等简单工具来收集网络、系统和服务的信息。

正如我们在上一个房间中了解到的那样，被动侦察可以让您在没有任何直接参与或联系的情况下收集有关目标的信息。您正在远距离观看或查看公开信息。

主动侦察要求您与目标进行某种接触。这种联系可以是打电话或在某种伪装下拜访目标公司，以此搜集更多的信息，通常是社会工程的一部分。或者，它可以直接连接到目标系统，无论是访问他们的网站还是检查他们的防火墙是否打开了 SSH 端口。把它想象成你正在仔细检查窗户和门锁。因此，必须记住，在获得客户签署的合法授权之前，不要从事主动侦察工作。

在这个房间里，我们专注于主动侦察。主动侦察始于与目标机器的直接连接。任何此类连接都可能在日志中留下信息，显示客户端 IP 地址、连接时间和连接持续时间等。但是，并非所有连接都是可疑的。可以让您的主动侦察显示为常规客户活动。考虑网页浏览；在数百名其他合法用户中，没有人会怀疑浏览器连接到目标网络服务器。当你作为红队（攻击者）的一部分工作时，你可以使用这些技术来发挥你的优势，并且不想惊动蓝队（防御者）。

在这个房间里，我们将介绍通常与大多数操作系统捆绑在一起的或很容易获得的各种工具。我们从 Web 浏览器及其内置的开发者工具开始；此外，我们还向您展示了如何 “武装” 网络浏览器以使其成为高效的侦察框架。之后，我们讨论其他良性工具，例如ping、traceroute 和 telnet 。所有这些程序都需要与目标建立联系，因此我们的活动将受到主动侦察。

任何想要熟悉基本工具并了解如何在主动侦察中使用它们的人都会对这个房间感兴趣。Web 浏览器开发者工具可能需要一些努力才能熟悉，尽管它提供了图形用户界面。此外，本房间所涵盖的命令行工具使用起来相对简单。

重要通知：请注意，如果您未订阅，AttackBox 将无法访问互联网，因此您需要使用 VPN 来完成需要访问互联网的问题。

网页浏览器

Web 浏览器可以是一个方便的工具，特别是它在所有系统上都很容易使用。您可以通过多种方式使用 Web 浏览器来收集有关目标的信息。

在传输层，浏览器连接到：

  • 通过 HTTP 访问网站时，默认使用 TCP 端口 80
  • 通过 HTTPS 访问网站时默认使用 TCP 端口 443

由于 80 和 443 是HTTP和 HTTPS 的默认端口，因此 Web 浏览器不会在地址栏中显示它们。但是，可以使用自定义端口来访问服务。例如，https://127.0.0.1:8834/ 将通过 HTTPS 协议在端口 8834 连接到 127.0.0.1（本地主机）。如果有 HTTPS 服务器侦听该端口，我们将收到一个网页。

在浏览网页时，您可以在 PC 上按 Ctrl+Shift+I 或在 Mac 上按Option + Command + I( ⌥ + ⌘ + I) 以打开 Firefox 上的开发者工具。类似的快捷方式也可以让您开始使用 Google Chrome 或 Chromium。开发者工具可让您检查浏览器已接收并与远程服务器交换的许多内容。例如，您可以查看甚至修改 JavaScript (JS) 文件、检查系统上设置的 cookie 并发现站点内容的文件夹结构。

Ping

Ping 应该会让您想起 ping-pong（乒乓球）游戏。你扔球并期望把它拿回来。ping 的主要目的是检查您是否可以到达远程系统以及远程系统是否可以返回您。换句话说，最初，这是用来检查网络连通性的；然而，我们更感兴趣的是它的不同用途：检查远程系统是否在线。

简单来说，ping 命令向远程系统发送数据包，远程系统回复。这样，您就可以断定远程系统在线并且两个系统之间的网络是正常工作的。

如果您喜欢更挑剔的定义，则 ping 是一个将 ICMP Echo 数据包发送到远程系统的命令。如果远程系统在线，并且 ping 数据包被正确路由并且没有被任何防火墙阻止，则远程系统应该发回 ICMP Echo Reply。同样，如果正确路由且未被任何防火墙阻止，ping 回复应该到达第一个系统。

此类命令的目的是在我们花时间执行更详细的扫描以发现正在运行的操作系统和服务之前确保目标系统在线。

在您的 AttackBox 终端上，您可以开始使用 ping： ping MACHINE_IP或ping HOSTNAME。在后者中，系统需要在发送 ping 数据包之前将 HOSTNAME 解析为 IP 地址。如果在Linux系统上不指定计数，则需要 输入CTRL+c 强制停止。因此，您可能会考虑 ping -c 10 MACHINE_IP 如果你只想发送十个数据包。在 MS Windows 系统上，这相当于 ping -n 10 MACHINE_IP 。

从技术上讲，ping 属于 ICMP（Internet 控制消息协议）协议。ICMP 支持多种类型的查询，但我们特别感兴趣的是 ping（ICMP echo/type 8）和 ping reply（ICMP echo reply/type 0）。使用 ping 不需要了解 ICMP 详细信息。

在以下示例中，我们已将数据包总数指定为 5。从 AttackBox 的终端，我们开始执行 ping 操作 MACHINE_IP 。我们了解到它 MACHINE_IP 已启动并且未阻止 ICMP 回显请求。此外，数据包路由上的任何防火墙和路由器也不会阻止 ICMP 回显请求。

在上面的示例中，我们清楚地看到目标系统正在响应。ping 输出表明它在线且可访问。我们已经发送了五个数据包，并且收到了五个回复。我们注意到，回复到达我们的系统平均需要 0.475 毫秒（毫秒），最大值为 0.636 毫秒。

从渗透测试的角度来看，我们将尝试发现更多关于这个目标的信息。我们将尝试尽可能多地了解，例如，哪些端口是开放的，哪些服务正在运行。

让我们考虑以下情况：我们关闭目标虚拟机，然后尝试 ping MACHINE_IP。正如您在以下示例中所期望的那样，我们没有收到任何回复。

在这种情况下，我们知道我们已经关闭了具有 MACHINE_IP 的目标计算机。对于每个 ping，我们使用的系统（在本例中为 AttackBox）都会以 “目标主机无法访问” 作为响应。我们可以看到我们发送了五个数据包，但没有收到，导致 100% 的数据包丢失。

一般来说，当我们没有收到 ping 回复时，有一些解释可以解释为什么我们没有收到 ping 回复，例如：

  • 目标计算机没有响应；可能仍在启动或关闭，或者操作系统已崩溃。
  • 它已从网络上拔下，或者路径上存在故障网络设备。
  • 防火墙被配置为阻止此类数据包。防火墙可能是在系统本身或单独的网络设备上运行的软件。请注意，默认情况下，MS Windows 防火墙会阻止 ping。
  • 您的系统已从网络上拔下。

追踪路由 Traceroute

顾名思义，traceroute 命令跟踪数据包从您的系统到另一台主机所采用的路由。traceroute 的目的是找到从您的系统到达目标主机时经过的跃点路由器的 IP 地址或数据包。该命令还显示了两个系统之间的路由器数量。它很有用，因为它指示您的系统和目标主机之间的跳数（路由器）。但是，请注意，由于许多路由器使用适应网络变化的动态路由协议，斯普哦数据包采用的路由可能会发生变化。

在Linux和 macOS 上，要使用的命令是traceroute MACHINE_IP，在 MS Windows 上，它是tracert MACHINE_IP. traceroute尝试发现从您的系统到目标系统的路径上的路由器。

没有直接的方法来发现从您的系统到目标系统的路径。我们依靠 ICMP 来 “欺骗” 路由器以泄露其 IP 地址。我们可以通过在 IP 标头字段中使用一个小的生存时间 (TTL) 来实现这一点。虽然TTL中的T代表时间，但TTL表示一个数据包在被丢弃之前可以通过的最大路由器/跳数；TTL 不是最大时间单位数。当路由器收到数据包时，它会将 TTL 减一，然后再将其传递给下一个路由器。下图显示，IP 数据包每经过一个路由器，其 TTL 值就会减 1。最初，它留给系统的 TTL 值为 64；经过4个路由器到达目标系统，TTL值为60。

但是，如果 TTL 达到 0，它将被丢弃，并且超过 ICMP 生存时间的消息将发送给原始发件人。在下图中，系统在发送到路由器之前将 TTL 设置为 1。路径上的第一个路由器将 TTL 减 1，导致 TTL 为 0。因此，该路由器将丢弃数据包并发送 ICMP 超时传输错误消息。请注意，某些路由器配置为在丢弃数据包时不发送此类 ICMP 消息。

在Linux 上，traceroute将首先在 TTL 为 1 的 IP 数据包中发送UDP数据报。因此，它会导致第一个路由器遇到 TTL=0 并发送回超过 ICMP 生存时间的信息。因此，TTL 为 1 将向您显示第一个路由器的 IP 地址。然后它将发送另一个 TTL=2 的数据包；该数据包将在第二个路由器处被丢弃。等等。让我们在实时系统上试试这个。

跟踪路由 B

在第二次运行 traceroute 程序时，我们注意到这次数据包走了更长的路线，经过了 26 个路由器。如果您正在对网络中的系统运行跟踪路由，则路由不太可能发生变化。然而，当数据包需要通过我们网络之外的其他路由器时，我们不能期望路由保持固定。

总结一下，我们可以注意到以下几点：

  • 您的系统和目标系统之间的跳数/路由器数取决于您运行 traceroute 的时间。无法保证您的数据包将始终遵循相同的路由，即使您在同一网络上或您在短时间内重复 traceroute 命令也是如此。
  • 一些路由器返回公共 IP 地址。您可能会根据预期渗透测试的范围检查其中一些路由器。
  • 一些路由器不返回回复。

远程登录 telnet

TELNET（电传网络）协议于 1969 年开发，用于通过命令行界面 (CLI) 与远程系统进行通信。因此，命令telnet使用 TELNET 协议进行远程管理。telnet 使用的默认端口是 23。从安全的角度来看，telnet以明文形式发送所有数据，包括用户名和密码。以明文形式发送可以让有权访问通信通道的任何人轻松窃取登录凭据。安全的替代方案是 SSH (Secure SHell) 协议。

然而，telnet 客户端由于其简单性，可以用于其他目的。我们知道 telnet 客户端依赖于TCP协议，所以您可以使用 Telnet 连接到任何服务并抓住它的 flag。使用telnet MACHINE_IP PORT，您可以连接到在TCP上运行的任何服务，甚至可以交换一些消息，除非它使用加密。

比方说，我们想要发现有关侦听端口 80 的 Web 服务器的更多信息。我们在端口 80 连接到服务器，然后使用 HTTP 协议进行通信。您无需深入研究 HTTP 协议；你只需要发出GET / HTTP/1.1. 若要指定默认索引页面以外的内容，您可以发出GET /page.html HTTP/1.1，这将请求page.html。我们还指定要使用 HTTP 1.1 版进行通信的远程 Web 服务器。要获得有效响应而不是错误，您需要为主机输入一些值host: example并按两次回车键。执行这些步骤将提供请求的索引页。

我们特别感兴趣的是发现已安装的网络服务器的类型和版本，Server: nginx/1.6.2. 在此示例中，我们与 Web 服务器通信，因此我们使用了基本的 HTTP 命令。如果我们连接到邮件服务器，我们需要根据协议使用适当的命令，例如 SMTP 和 POP3。

Netcat

Netcat 或简称 nc 具有不同的应用程序，这些应用程序对渗透测试人员来说可能具有很大的价值。Netcat 同时支持TCP和 UDP 协议。它可以作为连接到监听端口的客户端。或者，它可以充当服务器监听您选择的端口。因此，它是一种方便的工具，您可以将其用作 TCP 或 UDP 上的简单客户端或服务器。

首先，您可以像使用 Telnet 一样连接到服务器，使用 nc MACHINE_IP PORT 收集其 flag ，这与我们之前的telnet MACHINE_IP PORT 非常相似。请注意，您可能需要在 GET 行之后按 SHIFT+ENTER。

在上面显示的终端中，我们使用 netcat 连接到 MACHINE_IP 的 80 端口 nc MACHINE_IP 80。接下来，我们使用 GET / HTTP/1.1 向默认页面发出一个 get ；我们向目标服务器指定我们的客户端支持 HTTP 版本 1.1。最后，我们需要给我们的主机起一个名字，所以我们在一个新行上添加了host: netcat；你可以给你的主机起任何名字，因为这对这个练习没有影响。

根据 Server: nginx/1.6.2 我们收到的输出，我们可以知道在端口 80 上，我们有 Nginx 版本 1.6.2 侦听传入连接。

您可以使用 netcat 侦听 TCP 端口并连接到另一个系统上的侦听端口。

在服务器系统上，如果你想打开一个端口并监听它，你可以发出 nc -lp 1234 或更好，nc -vnlp 1234，它等同于nc -v -l -n -p 1234，就像你在Linux房间里记得的那样。字母的确切顺序无关紧要，只要端口号直接在前面-p。

注意：

  • 该选项-p应该出现在您要监听的端口号之前。
  • 该选项-n将避免DNS查找和警告。
  • 小于 1024 的端口号需要 root 权限才能监听。

总结

在这个房间里，我们介绍了许多不同的工具。很容易通过 shell 脚本将它们中的几个放在一起来构建一个原始的网络和系统扫描器。您可以使用traceroute映射到目标的路径，ping检查目标系统是否响应 ICMP Echo，并通过 telnet 尝试连接来检查哪些端口是开放的和可访问的。可用的扫描仪在更先进和复杂的水平上执行此操作，正如我们将在接下来的四个房间中看到的那样——nmap。

因为这些是基本工具，所以它们在大多数系统上都很容易获得。特别是，几乎每台计算机和智能手机上都安装了 Web 浏览器，它可以成为您武器库中的重要工具，用于在不发出警报的情况下进行侦察。如果你想对开发者工具有更深入的了解，我们推荐加入 Walking An Application。

在继续使用更复杂的工具之前，请确保您掌握我们在这个房间中介绍的不同的基本但必不可少的工具。
