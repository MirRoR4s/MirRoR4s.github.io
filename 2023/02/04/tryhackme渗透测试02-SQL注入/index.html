一. 前言

目标：了解如何检测和利用 SQL 注入漏洞

SQL（结构化查询语言）注入，通常称为 SQLi，是对 Web 应用程序数据库服务器的攻击，会导致执行恶意查询。 当 Web 应用程序使用未经正确验证的用户输入与数据库通信时，攻击者有可能窃取、删除或更改私人和客户数据，并攻击 Web 应用程序的身份验证方法以获取私有信息或客户区。 这就是为什么 SQLi 不仅是最古老的 Web 应用程序漏洞之一，而且也是最具破坏性的。

在这个房间里，您将了解什么是数据库、什么是 SQL 以及一些基本的 SQL 命令、如何检测 SQL 漏洞、如何利用 SQLi 漏洞以及作为开发人员如何保护自己免受 SQL 注入攻击。

二. SQL注入

2.1 什么是数据库？

如果您不习惯使用数据库或利用它们，可能需要习惯一些新术语，所以让我们从一些关于数据库的结构和工作方式的基础知识开始。

什么是数据库？

数据库是以有组织的方式以电子方式存储数据集合的一种方式。 数据库由 DBMS 控制，DBMS 是数据库管理系统的首字母缩写，DBMS 分为关系型和非关系型两个阵营，这个房间的重点将放在关系型数据库上，您会遇到的一些常见数据库是 MySQL、Microsoft SQL Server、Access、PostgreSQL 和 SQLite。 我们将在本任务结束时解释关系数据库和非关系数据库之间的区别，但首先，学习一些术语很重要。

在 DBMS 中，您可以有多个数据库，每个数据库都包含自己的一组相关数据。 例如，您可能有一个名为“商店”的数据库。 在此数据库中，您希望存储有关可供购买的产品的信息、已注册到您的在线商店的用户以及有关您收到的订单的信息。 您将使用称为表的东西将此信息单独存储在数据库中，每个表都用唯一的名称标识。 您可以在下图中看到这种结构，但您还可以看到企业如何使用其他单独的数据库来存储员工信息或客户团队。

什么是表？

表由列和行组成，一种有用的方式是将表想象成一个网格，列从左到右穿过顶部，包含单元格的名称，行从上到下，每一行都有 实际数据。

列：

每个列，更好地称为字段，每个表都有一个唯一的名称。 创建列时，您还可以设置它将包含的数据类型，常见的有整数（数字）、字符串（标准文本）或日期。 一些数据库可以包含更复杂的数据，例如包含位置信息的地理空间数据。 设置数据类型还可以确保不会存储不正确的信息，例如字符串“hello world”存储在用于日期的列中。 如果发生这种情况，数据库服务器通常会产生一条错误消息。 包含整数的列也可以启用自动递增功能； 这为每一行数据提供了一个唯一的数字，该数字随后续每一行而增长（递增），这样做会创建所谓的关键字段 key field ，关键字段对于每一行数据都必须是唯一的，可用于在 SQL 查询中查找确切的行。

行：

行或记录是包含各个数据行的内容。 当您向表中添加数据时，会创建新的行/记录，而当您删除数据时，会删除行/记录。

关系数据库与非关系数据库：

关系数据库将信息存储在表中，表之间通常共享信息，它们使用列来指定和定义所存储的数据，使用行来实际存储数据。表通常包含一个具有唯一ID(主键)的列，然后在其他表中使用该ID来引用它，并在表之间产生关系，因此称为关系数据库。

另一方面，非关系数据库有时被称为 NoSQL，它是不使用表、列和行来存储数据的任何类型的数据库，不需要构造特定的数据库布局，因此每一行数据都可以包含不同的信息，这可以比关系数据库提供更多的灵活性。这种类型的一些流行数据库是 MongoDB、Cassandra和 ElasticSearch。

现在您已经了解了数据库是什么，让我们学习如何使用SQL与它进行实际对话。

2.2 什么是 SQL？

SQL(结构化查询语言)是一种功能丰富的用于查询数据库的语言，这些SQL查询更好的称谓为“语句”。

我们将在这个任务中介绍的最简单的命令用于检索(选择)、更新、插入和删除数据。尽管有些相似，但一些数据库服务器有自己的语法和工作方式的轻微变化。所有这些例子都是基于MySQL数据库的。在学习了这些课程之后，您将能够轻松地为不同的服务器在线搜索替代语法。值得注意的是，SQL语法不区分大小写。

SELECT

我们将学习的第一个查询类型是用于从数据库检索数据的 SELECT 查询。

select * from users;

第一个单词SELECT告诉数据库我们想要检索一些数据，*告诉数据库我们想要从表中接收回所有列。例如，表可能包含三列(id、用户名和密码)。”from users”告诉数据库我们要从名为users的表中检索数据。最后，结尾的分号告诉数据库查询到此结束。

下一个查询与上面的查询类似，但这一次，我们不再使用*返回数据库表中的所有列，而是只请求用户名和密码字段。

select username,password from users;

下面的查询与第一个查询一样，通过使用*选择器返回所有列，然后使用“LIMIT 1”子句强制数据库只返回一行数据。将查询更改为“LIMIT 1,1”将强制查询跳过第一个结果，然后“LIMIT 2,1”将跳过前两个结果，以此类推。您需要记住，第一个数字告诉数据库希望跳过多少结果，第二个数字告诉数据库要返回多少行。

select * from users LIMIT 1;

最后，我们将使用where子句;这就是我们如何通过返回与特定子句匹配的数据来精细地挑选出我们需要的确切数据:

select * from users where username=’admin’;

这将只返回用户名等于 admin 的行。

select * from users where username != ‘admin’;

这将只返回用户名不等于 admin 的行。

select * from users where username=’admin’ or username=’jon’;

这将只返回用户名等于 admin 或 jon 的行。

select * from users where username=’admin’ and password=’p4ssword’;

这将只返回用户名等于 admin 且密码等于 p4ssword 的行。

使用 like 子句可以指定不是完全匹配的数据，而是通过选择百分号 % 表示的通配符的放置位置来指定以某些字符开头、包含或结尾的数据。

select * from users where username like ‘a%’;

这将返回用户名以字母 a 开头的所有行。

select * from users where username like ‘%n’;

这将返回用户名以字母n结尾的所有行。

这将返回用户名中包含字符 mi 的任何行。

INSERT语句告诉数据库，我们希望向表中插入一行新数据。”into users”告诉数据库我们希望将数据插入到哪个表中，”(username，password)”提供我们要为其提供数据的列，然后”values (‘bob’，’password’);”为前面指定的列提供数据。

UPDATE语句告诉数据库，我们希望更新表中的一行或多行数据。你可以使用”update %tablename% SET”来指定你想要更新的表，然后选择一个或多个你想要更新的字段，作为一个逗号分隔的列表，例如”username=’root’，password=’pass123’”，然后最后类似于select语句，你可以使用where子句，例如”where username=’admin;”来指定要更新的行。

2.4 In-Band SQLi 带内 SQL注入

带内SQL注入

带内 SQL 注入是最容易检测和利用的类型;带内通信只是指使用相同的通信方法来利用漏洞并接收结果，例如，在网站页面上发现SQL注入漏洞，然后能够从数据库中提取数据到同一页面。

基于错误的SQL注入
这种类型的 SQL 注入对于轻松获取关于数据库结构的信息是最有用的，因为来自数据库的错误消息会直接打印到浏览器屏幕上。这通常可用于枚举整个数据库。

基于联合的SQL注入
这种类型的注入利用 SQL UNION 操作符和 SELECT 语句一起向页面返回额外的结果。这种方法是通过 SQL 注入漏洞提取大量数据的最常用方法。

盲注

与带内SQL注入不同，在带内SQL注入中，我们可以直接在屏幕上看到攻击的结果，盲注是当我们几乎没有反馈来确认注入的查询是否成功，这是因为错误消息已经被禁用，但注入仍然可以工作。您可能会感到惊讶，我们只需要一点点反馈就可以成功枚举整个数据库。

2.8 Out-of-Band SQLi

带外SQL注入并不常见，因为它要么依赖于数据库服务器上启用的特定功能，要么依赖于web应用程序的业务逻辑，后者基于SQL查询的结果进行某种外部网络调用。

带外攻击通过具有两个不同的通信通道进行分类，一个用于发起攻击，另一个用于收集结果。例如，攻击通道可以是一个web请求，而数据收集通道可以监视向您控制的服务发出的HTTP/DNS请求。

1)攻击者使用注入有效载荷向易受SQL注入攻击的网站发出请求。

2)本网站对数据库进行SQL查询，该数据库也传递了黑客的有效载荷。

3)有效载荷包含一个请求，该请求迫使一个HTTP请求返回黑客的机器，其中包含数据库中的数据。

2.9 修复SQL注入

修复

由于SQL注入漏洞的影响，开发人员确实有一种方法来保护他们的web应用程序，通过遵循以下建议:

准备语句(带有参数化查询):

在准备好的查询中，开发人员编写的第一件事是SQL查询，然后将任何用户输入作为参数添加。编写准备好的语句可以确保SQL代码结构不会改变，并且数据库可以区分查询和数据。作为一个好处，它还使您的代码看起来更干净，更容易阅读。

输入验证:

输入验证对于保护放入SQL查询的内容有很大帮助。使用允许列表可以将输入限制为某些字符串，或者编程语言中的字符串替换方法可以过滤希望允许或不允许的字符。

转义用户输入:

允许用户输入包含’ “ $ \这样的字符会导致SQL查询中断，更糟糕的是，正如我们已经了解到的那样，会使它们成为注入攻击的对象。转义用户输入是在这些字符前加上反斜杠()的方法，这将导致它们作为常规字符串而不是特殊字符进行解析。
