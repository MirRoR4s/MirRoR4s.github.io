网络安全中的职业

了解网络安全领域的不同职业。

1. 介绍

网络安全职业的需求量越来越大，薪水也越来越高。安全行业有许多不同的工作，从进攻性渗透测试（黑客攻击机器和漏洞报告）到防御性安全（防御和调查网络攻击）。

为什么要从事网络职业：

  • 高薪 - 安全工作起薪高
  • 令人兴奋的 - 工作可以包括合法的黑客系统或防御网络攻击
  • 供不应求——有超过 350 万个网络职位空缺

这个房间通过提供有关各种网络安全角色的信息来帮助您进入网络安全领域；它还链接到不同的学习路径，您可以使用这些路径开始培养您的网络技能。

2. 安全分析师

安全分析是负责维护组织数据的安全性

安全分析师是跨组织构建安全措施以保护公司免受攻击不可或缺的一部分。分析师探索和评估公司网络，以发现可操作的数据和建议，供工程师制定预防措施。此工作角色需要与各种利益相关者合作，以了解安全要求和安全形势。

职责

  • 与各利益相关者合作，分析整个公司的网络安全
  • 编制有关网络安全的持续报告，记录安全问题和采取的应对措施
  • 制定安全计划，结合对新攻击工具和趋势的研究，以及跨团队维护数据安全所需的措施。

3. 安全工程师

安全工程师负责设计、监控和维护安全控制、网络和系统，以帮助防止网络攻击。

安全工程师使用威胁和漏洞数据开发和实施安全解决方案——通常来自安全团队成员。安全工程师致力于规避各种攻击，包括 Web 应用程序攻击、网络威胁以及不断发展的趋势和策略。最终目标是保留并采用安全措施来减轻攻击和数据丢失的风险。

职责

  • 跨软件测试和筛选安全措施
  • 监控网络和报告以更新系统和缓解漏洞
  • 确定并实施最佳安全性所需的系统

学习路径

TryHackMe 的学习路径将为您提供基础技术知识和实践经验，这对于成为一名成功的安全工程师至关重要。

  • Cyber Defense
  • JR Penetration Tester
  • Offensive Pentesting

4. 事件响应者 Incident Responder

事件响应者负责在攻击者的操作仍在展开时识别并减轻攻击。

事件响应者对安全漏洞做出富有成效和高效的响应。职责包括为组织制定计划、政策和协议，以便在事件发生期间和事件发生后执行。这通常是一个压力很大的位置，随着攻击的展开，需要实时进行评估和响应。事件响应指标包括 MTTD、MTTA 和 MTTR - 检测、确认和恢复（从攻击中）的时间。目的是实现快速有效的响应，保持财务状况并避免负面的违规影响。最终，事件响应者保护公司的数据、声誉和财务状况免受网络攻击。

职责

  • 制定并采用全面、可操作的事件响应计划
  • 保持强大的安全最佳实践并支持事件响应措施
  • 事件后报告和为未来攻击做准备，考虑从事件中吸取教训和适应

学习路径

TryHackMe 的学习路径将为您提供基础技术知识和实践经验，这对于成为一名成功的事件响应者至关重要。

  • Cyber Defense

5. 数字取证审查员 Digital Forensics Examiner

数字取证审查员负责使用数字取证调查事件和犯罪

如果你喜欢扮演侦探，这可能是完美的工作。如果您是执法部门的一员，您将专注于收集和分析证据以帮助破案：指控有罪者和无辜者无罪。另一方面，如果您的工作属于保护公司网络的范畴，您将使用您的取证技能来分析事件，例如违反政策的行为。

职责

  • 在遵守法律程序的同时收集数字证据
  • 分析数字证据以找到与案件相关的答案
  • 记录您的发现并报告案例

6. 恶意软件分析师 Malware Analyst

恶意软件分析师负责分析所有类型的恶意软件以更多地了解它们的工作原理和作用

恶意软件分析师的工作包括分析可疑程序、发现它们的行为并编写有关其发现的报告。恶意软件分析师有时被称为逆向工程师，因为他们的核心任务围绕着将编译的程序从机器语言转换为可读代码，通常是低级语言。这项工作要求恶意软件分析师具有很强的编程背景，尤其是汇编语言和 C 语言等低级语言。最终目标是了解恶意程序执行的所有活动，了解如何检测和报告它。

职责

  • 对恶意程序进行静态分析，这需要进行逆向工程
  • 通过观察受控环境中的活动对恶意软件样本进行动态分析
  • 记录并报告所有发现

7. 渗透测试员 Penetration Tester

渗透测试员负责测试技术产品的安全漏洞

您可能会看到渗透测试被称为渗透测试和道德黑客攻击。渗透测试人员的工作职责是测试公司内部系统和软件的安全性——这是通过尝试通过系统化的黑客攻击来发现缺陷和漏洞来实现的。渗透测试人员利用这些漏洞来评估每个实例的风险。然后，公司可以利用这些见解来纠正问题，以防止现实世界中的网络攻击。

职责

  • 对计算机系统、网络和基于 Web 的应用程序进行测试
  • 执行安全评估、审计和分析策略
  • 评估和报告洞察力，推荐攻击预防措施

8. 红队人员 Red Teamer

红队人员负责扮演对手的角色，攻击组织并从敌人的角度提供反馈

红队与渗透测试员有相似之处，具有更有针对性的工作角色。渗透测试人员希望发现系统中的许多漏洞，以保持网络防御的良好状态，而红队则负责测试公司的检测和响应能力。该工作角色需要模仿网络罪犯的行为、模拟恶意攻击、保留访问权限并避免被发现。红队评估最多可以运行一个月，通常由公司外部的团队进行。它们通常最适合拥有成熟安全程序的组织。

职责

  • 模拟威胁行为者的角色，以发现可利用的漏洞、保持访问权限并避免被发现
  • 评估组织的安全控制、威胁情报和事件响应程序
  • 评估和报告洞察力，为公司提供可操作的数据以避免真实世界的实例

学习路径
