1.1.16

当$x$是一个整数的时候，$[-x]=-x$,所以$-[-x]=x$是大于或等于x的最小整数。当$x$不是一个整数的时候，令$x=a+r$，$a$是$x$的整数部分，$r$是$x$的小数部分，$0<r<1$,此时$[-x]=[-a-r]=-a+[-r]=-a-1,-[-x]=a+1$是大于或等于$x$的最小整数

1.1.17

题目应该默认$x$是一个实数，我们令$x=a+r$，其中$a$是$x$的整数部分，$r$是$x$的小数部分，最接近$x$的整数应该是$[x]$或者$[x]+1$

当$0<=r<1/2$，此时最接近$x$的整数应该是$[x]=[a+r]=a+[r]=a$,并且$[x+1/2]=[a+r+1/2]=a+[r+1/2]=a$，所以$[x]=[x+1/2]$，证明成立。

当$1/2<=r<1$，此时最接近$x$的整数应该是$[x]+1$，所以只需证明$[x]+1=[x+1/2]$即可。$[x+1/2]=[a+r+1/2]=a+[r+1/2]=a+1$，而$[x]+1=[a+r]+1=a+[r]+1=a+1$，所以$[x+1/2]=[x]+1$，综上所述证明成立，即$[x+1/2]$是最接近$x$的最大的整数

1.1.19

当$x=0$的时候，证明显然成立，所以只需要证明$x$是正实数的情况即可。

令$x=k+r$，$k$是$x$的整数部分，$r$是$x$的小数部分，$0<=r<1$。将$k$表示为$k=a^2+b$，其中$a$是使得$a^2<=k$的最大整数。所以$a^2<=k=a^2+b<=x=a^2+b+r<(a+1)^2$，（这里存疑，没有清楚地明白为什么$(a+1)^2 > a^2+b+r$）。根据上式可以知道$a<=\sqrt{x}<(a+1)$，所以$[\sqrt{x}]=a$。

而$[x]=[a^2+b+r]=a^2+b$，所以$ a<=\sqrt{[x]}=\sqrt{a^2+b}<a+1$

所以$[\sqrt{[x]}]=a=[\sqrt{x}]$。

pickle反序列化初探

基本知识

pickle简介

与PHP类似，python也有序列化功能以长期储存内存中的数据。pickle是python下的序列化与反序列化包。

与json相比，pickle以二进制形式存储，不易人工阅读；json可用跨语言，而pickle是python专用的；pickle能表示python几乎所有的类型，包括自定义类型；json只能表示一部分内置类型且不能表示自定义类型。

pickle实际上可用看作一种独立的语言，通过对opcode的更改编写可以执行的python代码，覆盖变量等操作。直接编写的opcode灵活性比使用pickle序列化生成的代码更高，有的代码不能通过pickle序列化得到（pickle解析能力大于pickle生成能力）。

pickle库及函数

pickle是python语言的一个标准模块，实现了基本的数据序列化和反序列化。pickle模块是以二进制的形式序列化后保存到文件中，保存文件的后缀为.pkl，不能直接打开进行预览。

object.__reduce__()函数

反序列化漏洞出现在__reduce__()魔法函数上，这一点和PHP中的__wakeup()魔术方法类似，都是因为每当反序列化过程开始或者结束时，都会自动调用这类函数。而这恰好是反序列化漏洞经常出现的地方。

而且在反序列化过程中，因为编程语言需要根据反序列化字符串去解析出自己独特的语言数据结构，所以就必须在内部把解析出来的结构去执行一下。如果在反序列化过程中出现问题，便可能直接造成RCE漏洞。

在python2中只有内置类才有__reduce__方法，即用class A(object)声明的类，而python3中已经默认都是内置类了

在开发时，可以通过重写类的object.__reduce__()函数，使之在被实例化时按照重写的方式进行。具体而言，python要求object.__reduce__()返回一个(callable,([para1,para2…])[,…])的元组，每当该类的对象被unpickle时，该callable就会被调用以生成对象(该callable其实是构造函数)。

在pickle的opcode中，R的作用与object.__reduce__()关系密切：选择栈上的第一个对象作为函数、第二个对象作为参数（第二个对象必须为元组），然后调用该函数。其实R正好对应object.__reduce__()函数，object.__reduce__()的返回值会作为R的作用对象，当包含该函数的对象被pickle序列化时，得到的字符串是包含了R的。

pickle过程详细解读

pickle解析依靠Pickle Virtual Machine（PVM）进行，

PVM设计到三个部分：1.解析引擎 2.栈 3.内存

解析引擎：从流中读取opcode和参数，并对其进行解释处理。重复这个动作，直到遇到 . 停止。最终留在栈顶的值将被作为反序列化对象返回。

栈：由Python的list实现，被用来临时存储数据、参数以及对象。

memo：由Python的dict实现，为PVM的生命周期提供存储。说人话：将反序列化完成的数据以key-value的形式存储在memo中，以便后来使用。

如何手写opcode

在CTF中，很多时候需要一次执行多个函数或一次进行多个指令，此时就不能光用__reduce__来解决问题，因为__reduce__一次只能执行一个函数，当exec被禁用时，就不能一次执行多条指令了；所以我们需要手动拼接或构造opcode，手写opcode是pickle反序列化比较难的地方。

在这里可以体会到为何pickle是一种语言，直接编写的opcode灵活性比使用pickle序列化生成的代码更高，只要符合pickle语法，就可以进行变量覆盖、函数执行等操作。

根据前文不同版本的opcode可以看出，版本0的opcode更方便阅读，所以手动编写时，一般选用版本0的opcode。下文中都是版本为0的opcode。

由这些opcode我们可以得到一些需要注意的地方：

编写opcode时要想象栈中的数据，以正确使用每种opcode。

在理解时注意与python本身的操作对照，比如python列表的append对应a、extend对应e；字典的update对应u。

c操作符会尝试import库，所以在pickle.loads时不需要漏洞代码中先引入系统库。

pickle不支持列表索引、字典索引、点号取对象属性作为左值，需要索引时只能先获取相应的函数，不如getattr、dict.get才能进行。但是因为存在s、u、b操作符，作为右值是可以的；即查值不行，赋值可以。pickle能够索引查值的操作只有c、i。而如何查值也是CTF的一个重要考点。

s、u、b操作符可以构造并赋值原来没有的属性、键值对。

CTF实战

做题之前：了解pickle.Unpickler.find_class()

由于官方针对pickle的安全问题的建议是修改find_class()，引入白名单的方式来解决，很多CTF题都是针对该函数进行，所以搞清楚如何绕过该函数很重要。
什么时候会调用find_class()：

  1. 从opcode角度看，当出现c、i、b'\x93'时，会调用，所以只要在这三个opcode直接引入模块时没有违反规则即可。

4.1

目前使用的很多对称分组加密算法都是基于Feistel分组密码结构。因此，对Feistel结构的研究揭示了这些最新密码背后的原理。

4.2

流密码是对数字数据流一次加密一个比特或一个字节的密码。分组密码是将一个明文块作为一个整体来处理，并用它来产生一个长度相等的密文块。

4.3

如果使用一个较小的块大小，例如n = 4，那么该系统相当于一个经典的替换密码。对于小的分组块，这样的系统很容易对明文进行统计分析。对于较大的块，密钥的大小是$n * 2^n$的量级，这使得系统不切实际。

4.4

乘积密码是依次使用两个或两个以上基本密码，所得结果的密码强度将强于所有单个密码的强度。

4.5

扩散的方法是尽可能地使明文和密文间的统计关系变得复杂，以挫败推导出密钥的企图。混淆则是尽可能使密文和加密密钥间的统计关系更加复杂，以阻止攻击者发现密钥。

在扩散过程中，明文的统计结构被消散到密文当中去。这是通过让每个明文数字影响许多密文数字的值来实现的，这相当于说每个密文数字受许多明文数字的影响。混淆试图使密文的统计数据和加密密钥的值之间的关系尽可能复杂，以再次阻止发现密钥的尝试。因此，即使攻击者能够在一定程度上掌握密文的统计信息，但使用密钥生成密文的方式是如此复杂，以至于很难推断出密钥。这是通过使用复杂的替换算法实现的。

4.6

分组长度：更大的分组长度意味着更高的安全性（），但减少了加解密的速度。

密钥长度：更大的密钥长度意味着更高的安全性，但是也许会减少加解密的速度

迭代轮数：Feistel密码的本质是一个简单的单轮迭代只能提供不充分的安全性，但是多轮迭代可以增加安全性

子密钥产生算法：这个算法的复杂度越高，应该导致密码分析越困难。

轮函数：同样地，更复杂的轮函数通常意味着抵抗密码分析的能力越强。

快速的软件加密/解密：在许多案例中，加密被嵌入到应用或应用程序的函数当中去，以此替代硬件实现加密/解密的方式。因此，算法执行的速度就成为了一个问题。

易于分析：尽管我们想要我们的算法在密码分析上尽可能地困难，但是使算法易于分析也有许多的好处。例如，如果算法可以被简洁清晰地解释，那么就更容易对算法进行密码漏洞分析，因此对其强度的保证程度更高。

雪崩效应是任何加密算法的一种特性，明文或密钥的微小变化都会导致密文的显著变化

此算法名为Cocks algorithm

a. Cocks利用了中国剩余定理，该定理说，可以用整数的余数对一组相对素数模取模，重构一定范围内的整数。特别是对于质数相对较多的P和Q, 0 < M < N范围内的任何整数M都可以是M mod P和M mod Q的一对数字，并且在给定M mod P和M mod的情况下，有可能恢复M.安全性在于很难找到N的质因数。

b. 在RSA中，用户构造一对整数d和e，使得$d\times e \equiv 1 \mod ((P-1)(Q-1))$，然后公布e和N作为公钥。

Cocks是一种特殊情况，其中e=N。

c. RSA算法具有对称的优点;加密和解密都使用相同的过程，这简化了所需的软件。此外，还可以任意选择e，以便使用特别简单的版本进行公钥加密。

d. 私钥k是P和Q；公钥x是N；明文p是M；密文z是C。M1由k的两部分相乘得到。M2是由M的N次幂模N组成。M3由本题描述中的过程组成

$a^{2c}\mod n = (a^c)^2 \mod n ,if \space b_i =0$

$a^{2c+1}\mod n = a \times (a^c)^2 \mod n ,if \space b_i = 1$

该算法保持不变，即$d = a^c mod n$，当c增加2倍和递增时，直至$c = b$。现在让我们考虑问题中的算法，它改编自[KNUT98, 462页]中的算法。该算法从右到左(最低位到最高位)处理b的二进制表示。在这种情况下，算法保持不变，$a^n=d\times T^E$。最后，$E = 0$，剩下$a^n = d$。
