The Carol Corruptor#

题目描述#

Krampus is back at it again! He just activated his new creation, Krampto, the carol corruptor. A spy reindeer noticed that to stop the machine, we need a secret code.

Can you help Santa stop Krampto?

题目内容#

import random
from carol_corrupter import FLAG, corrupt_carol


class Krampto:
    def __init__(self):
        self.__SECRET = ""
        self.__generate_secret()

    def __generate_secret(self):
        # is this enough?! maybe... maybe not.
        for i in range(31):
            print(random.getrandbits(640))
        self.__SECRET = str(random.getrandbits(2048))[:32]

    def stop_machine(self):
        proposed_secret = input("SECRET: ")
        if proposed_secret == self.__SECRET:
            print(FLAG)
            exit()
        print("Nuh uh, carols are now forever corrupted!")
        exit()

    def corrupt_carol(self):
        corrupted_carol = corrupt_carol()
        for part in corrupted_carol:
            print(part)


if __name__ == "__main__":
    krampto = Krampto()
    options = """
    1. Corrupt Carol
    2. Stop Machine
    3. Quit
    """
    while True:
        print(options)
        option = input("> ")
        if option == "1":
            krampto.corrupt_carol()
        elif option == "2":
            krampto.stop_machine()
        else:
            exit()

题目分析#

输入 2 猜中 secret 就输出 flag。连接到服务器之后会给我们 31 个 640 比特的随机数，所以题目的考点应该是 random 模块伪随机数预测。我们手里有 $31640 = 19840$ ，但我们需要 $62432=19968$ 比特。也就是说我们少了4个32比特的数，这里可以使用 z3 模块来求解。github上有相关的脚本。

相关链接：

1.完整WP

2.z3模块求解剩余4个32字节整数脚本

3.python的伪随机数回溯脚本，用于退回之前的状态

解题脚本#

from pwn import *
from Untwister import *
from Crypto.Util.number import *
from extend_mt19937_predictor import ExtendMT19937Predictor
def trans(data):
    Num = []

    for i in data:
        while i != 0:
            Num.append(0xFFFFFFFF & i)
            i >>= 32

    assert len(Num) == 620
    return Num
def predict(data):
    t = Untwister()
    for i in data:
        t.submit(bin(i)[2:])
    t.submit('?' * 32)
    t.submit('?' * 32)
    t.submit('?' * 32)
    t.submit('?' * 32)

    r = t.get_random() # 不出意外，到这里就预测成功啦！

    predictor = ExtendMT19937Predictor()

    for _ in range(624):
        predictor.setrandbits(r.getrandbits(32), 32)

    #开始时空回溯，退回到
    _ = [predictor.backtrack_getrandbits(32) for _ in range(624)]
    predictor.backtrack_getrandbits(32 * 4)
    secret_guess = str(predictor.predict_getrandbits(2048))[:32]
    return secret_guess


def send_recv(ip,port):
    r = remote(ip,port)
    data = []
    for i in range(31):
        tmp = int(r.recvline().strip().decode())
        print(tmp)
        data.append(tmp)
    assert len(data) == 31
    #此时接收到了 31 个 640 比特的整数，要转化成 32 比特整数的形式
    data = trans(data) #现在有 620 个 32 比特的整数，接下来需要调用 Untwister 求出剩下的 4 个整数

    flag = predict(data)
    r.sendlineafter(b"> ",b"2")
    r.sendlineafter(b"SECRET: ",str(flag).encode())
    print(r.recvline().strip())







if __name__ == "__main__":

    ip = "challs.htsp.ro"
    port = "10003"
    send_recv(ip,port)
离谱的点#

看到一个别人的答案是：

from pwn import *
from mt19937predictor import MT19937Predictor

predictor = MT19937Predictor()

r = remote("challs.htsp.ro", 10003)

for i in range(31):
    predictor.setrandbits(int(r.recvline().decode()), 640)

secret = str(predictor.getrandbits(2048))[:32]

r.clean()

r.sendline(b'2')
r.clean()
r.sendline(str(secret).encode())
print(r.recvline().strip())

明明只往 predictor 里面灌输了 $640*31$ 个比特的数字，但是还是成功预测出来了。我看了一下mt19937predictor 的源码，里面也写道了需要 624 个连续的 DOWRDs 才可以完全唯一确定内部状态。

PolyEq#

题目描述#

As you already know, Santa is a math fanatic. This year he spent his summer studying interesting polynomial equations. He found a very difficult problem and he solved it with some clever maths. Now he proposes the same problem for you to solve:

Let f and g be two monic polynomials of degree 32 over Zn.
Let S = {x1, x2, …, xt} be a set of 2D points in ZnXZn. You know that following equation holds for at least V points:
f(x) / g(x) = y (1)

Find two polynomials f and g such that equation (1) holds for at least V points.

Note: for every point in the set of points that hold for equation (1), g(x) is invertible.

题目内容#

题目分析#

题目让我们找到两个多项式 f 和 g，使得 f(x)/g(x) = y 对 某个点集 S 中的 V 个点成立。

f 和 g 是定义在 $Z_n$ 上的 次数为 32 的两个首一多项式，并且题目会将点集 S 告诉我们，同时还会告诉我们 V 的数值。

so massive spoilers ahead for PolyEq / Interpolation with Errors from NSUCRYPTO
Observation 1:
The first thing is that if there would be a small amount of points that are invalid (let's say 1024 are correct out of 1100) we could randomly sample 64 points and hope that those points are from the valid set of points (if the number of invalid points is small enough this will happen with quite a big probability). So we would like to somehow filter the set of points such that instead of 3072 total points we would have a number way closer to 1024 
Observation 2:
Next we can observe that N is not prime it is 2 * 3 * 5 * 1327. What is interesting about it? well a polynomial that is valid in Z_n is also valid in Z_2, Z_3 and Z_5
Observation 3:
So now how can we filter the points? well if you have two points (x, y1) and (x, y2), they can't be both valid points, since f(x)/g(x) is not a function. But the probability of this happening in Z_n is very small so it's not interesting
Observation 4:
Now we can combine the last two observation, we can do the filtering over Z_2, Z_3 and Z_5. we split the big set of points into many smaller sets of points that respect the condition in Obs 3
Observation 5:
We then interset all those filtered groups and keep only the ones bigger than 1024 (since we know our group will have at leas 1024 points), finally we brute force with Obs 1 on this filtered set
