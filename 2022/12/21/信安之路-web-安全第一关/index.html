web运行基础环境搭建#

任务目标#

准备学习环境，学习 web 服务器的搭建过程，并做相应的加固学习

主机环境#

ubuntu+nginx+php-fpm+mysql

报告要求#

将整个环境的搭建过程进行详细记录，收集网络上的加固文档，学习加固技术，从而思考不加固可能存在的安全问题，对于加固的过程以及对于安全的思考都需要做详细的记录。

最终目标#

能够运行 php 代码并且可以使用 php 连接 mysql，成功执行 mysql 的语句

LNMP环境搭建#

安装 nignx#

sudo apt-get update
sudo apt-get install nginx -y
sudo service nginx start

安装好之后进行配置

cd /etc/nginx/sites-available/
sudo cp default default.bak
sudo vim default

使用如下命令检查配置文件的语法错误：

sudo nginx -t

准备好后，重新加载 nginx 的配置文件

sudo systemctl reload nginx

安装 MySQL#

sudo apt-get install mysql-server mysql-client

安装好了之后在进入 MySQL 服务器要输入密码，但是一开始并没有设置密码。看网上说似乎是 MySQL会随机设置一个密码，最后在这个链接找到了解决办法：Ubuntu系统忘记mysql root密码

之后使用如下命令拿着设置的密码进入mysql服务。

mysql -u root -p

安装 PHP#

与Apache不同，Nginx不包含本地PHP处理。为此，我们必须安装PHP-FPM（FastCGI Process Manager）。FPM是另一种PHP FastCGI实现，具有一些对重载站点有用的附加功能。另外还需要安装php-mysql以允许PHP与MySQL数据库通信。

选择适合自己操作系统的php版本安装，我的好像只有8.1版本。

sudo apt-get install php8.1-fpm php8.1-mysql -y

修改Nginx默认配置#

黑客经常抓包分析我们服务器的响应头，而很多服务器并没有对返回的响应头做一个过滤，这会泄露很多我们服务器的信息，比如服务器类型、版本号等。所以现在我们修改一下 nginx 的配置文件，将一些不安全的HTTP响应头删掉。

http{
	server_tokens off;	# 隐藏 Nginx 版本号
	autoindex off;	# 关闭目录访问
    limit_conn addr 100;	# 给定的key设置最大连接数，这里key是addr
    limit_rate 100k;	# 对每个连接限速100k
    
# 控制缓冲区大小
client_body_buffer_size 1K;
client_header_buffer_size 1k;
client_max_body_size 1k;
large_client_header_buffers 2 1k;



# 清除不安全的HTTP响应头
more_clear_headers "X-Powered-By";
more_clear_headers "Server";
more_clear_headers "ETag";
more_clear_headers "Connection";
more_clear_headers "Date";
more_clear_headers "Accept-Ranges";
more_clear_headers "Last-Modified";

# 禁止不安全的 HTTP 方法
if($request_method !~ ^(GET|HEAD|POST)$){
    retrun 444;
}
UFW#

ufw是一个主机端的iptables类防火墙配置工具，比较容易上手。一般桌面应用使用ufw已经可以满足要求了。我们可以通过配置防火墙，允许或者禁止一些特定端口的进出，比如不允许外来的IP连接我们，只允许我们连接别人等。

安装

sudo apt-get install ufw

应用配置：

大部分应用都附带一份应用配置，它描述了服务，并且包含了 UFW 设置。

sudo ufw app list

启用 UFW，允许SSH远程连接：

sudo ufw allow ssh
