jvm学习总结

发表于 2022-10-18 分类于 JAVA

我心血来潮忽然觉得不是很理解jvm的相关概念和知识，于是打算书写这篇文章做一个汇总。

定义：什么是JVM？#

JVM是Java Virtual Machine的缩写，即java虚拟机。它是一个虚构出来的计算机，一种规范。通过在实际的计算机上仿真模拟各类计算机功能实现。简单地说，JVM类似于一台运行在windows或linux这些操作系统环境下的小电脑，它直接和操作系统进行交互，不与硬件直接交互，但是可以操作系统帮助我们完成和硬件的交互工作。

上图可以看到，操作系统是JVM和硬件沟通的桥梁。

Java文件是如何被运行的？#

比如我们现在书写了一个HelloWorld.java文件，抛开该文件的所有东西不谈，其就类似于一个文本文件，只不过这个文本文件写的都是英文，而且有一定的缩进而已。

类加载器#

但我们的JVM是不认识文本文件的，所以它需要一个编译，让其称为一个它会读二进制文件的HelloWorld.class。但是如果想让JVM执行这个.class文件，那么我们就需要将其装进一个类加载器中，这个加载器就像一个搬运工一样，会把所有的.class文件全部搬进JVM里面来。

JVM五大组成部分#

方法区#

方法区用于存放类似于元数据信息方面的数据，比如类信息，常量，静态变量，编译后代码。类加载器将.class文件搬过来就是先丢到这一块上。

堆#

堆主要存放了一个存储的数据，比如对象实例，数组等，它和方法区都同属于线程共享区域。也就是说它们都是线程不安全的。

栈#

栈是我们的代码运行空间，我们编写的每一个方法都会放到栈里面执行。

程序计数器#

主要就是完成一个加载工作，类似于一个指针一样的，指向下一行我们需要执行的代码。和栈一样，都是线程独享的，就是说每一个线程都会有自己对应的一块区域而不会存在并发和多线程的问题。

小总结#

1、Java文件经过编译后变成.class字节码文件

2、字节码文件通过类加载器被搬运到JVM虚拟机中

3、虚拟机主要的5大块：方法区，堆都为线程共享曲线，有线程安全问题，栈和本地方法栈和计数器都是独享区域，不存在线程安全问题，而JVM的调优主要就是围绕堆、栈两大块进行。
