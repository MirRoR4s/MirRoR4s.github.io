Determination is key#

题目描述#

I’ve written a fancy function to generate my own primes, I don’t trust these government-owned prime-generation algorithms these days, you know… Is this secure enough?

题目内容#

分析#

生成一个位于 [2,2**16] 区间的整数，并将此整数的每一位乘在一起得到乘积 r。如果 r 为 0 或 1 跳过此次循环，如果 r 不是素数或者 小于 2 的 256 次方则 令 $r=2r-1$。

我们直接运行这个素数生成算法，然后查看 模数是否整除我们生成的素数就可以了。根据素数分布定理，这个过程应该不会太久。

Magic, not crypto#

题目描述#

I’ve used my magic powers to obfuscate the challenge, will you be able to revert it?
nc tcp.challs.m0lecon.it 9839

题目内容#

分析#

题目将 flag 进行 20次 加密，其中 RSA 加密至多使用一次，其余加密操作是 base64 编码、rot13 编码、逆序。

题目已告知所用的加密算法步骤以及密文，我们简单地写出其对应的解密算法即可。注意到其中 RSA算法解密脚本的编写有些困难，因为我们甚至不知道模数n的值，但是观察源码就会发现 p、q的生成由明文的前四个字节决定

只要拿到RSA算法输入的前四个字符就可以获得p、q，这是因为种子固定那么生成的数字也固定。

​ 只要我们不断重连服务器直至第一次使用的加密算法是RSA就可以停下来，此时RSA算法的输入的前四个字节是 ptm{，亦或者说加密算法根本就没有使用 RSA，此时也可以轻松解出来！
