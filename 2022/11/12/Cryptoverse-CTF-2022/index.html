RSA 加密的安全性取决于分解模数 N 的困难性，幸运地是题目的模数已经在 factordb 的数据库中，所以我们可以直接分解得到 p、q。

from Crypto.Util.number import *

n = 0x7c05a45d02649367ebf6f472663119777ce5...
e = 0x10001
ct = 0x35b63f7513dbb828800a6bcd708d87a6c9f33af63...
p = 8156072525389912369788197863285751656042...      #from factordb
q = n // p
assert (p * q == n)

d = inverse(e, (p-1) * ( q-1))
m = pow(ct, d, n)
print(long_to_bytes(m))

RSA2

题目描述：

Too much information was leaked in RSA 1. See if this one is harder to break.

题目源码：

from Crypto.Util.number import inverse, bytes_to_long, getPrime, isPrime
from math import gcd
from secret import flag

PBITS = 512
e = 0x10001

def stage_one(data: bytes):
    m = bytes_to_long(data)
    p = getPrime(PBITS)
    q = getPrime(PBITS)
    b = 7
    n = p**b * q
    print(f"p = {p}")
    print(f"e = {e}")
    print(f"dp = {inverse(e, p-1)}")
    print(f"b = {b}")
    print(f"ct = {pow(m, e, n)}\n")

def stage_two(data: bytes):
    m = bytes_to_long(data)
    p = getPrime(PBITS)
    q = p + 2
    while not isPrime(q):
        q += 2
    n = p * q
    print(f"n = {n}")
    print(f"e = {e}")
    print(f"ct = {pow(m, e, n)}\n")

print("=== Stage 1 ===")
stage_one(flag[:len(flag)//2])
print("=== Stage 2 ===")
stage_two(flag[len(flag)//2:])

分析

stage 1

糟糕的点在于题目没有将模数 N 告诉我们，这时候只能从大小入手。由于p是 64 字节，所以 $p^7$ 大约是 450 字节，我们猜测 flag 的字节长度肯定没有这么大，所以极有可能密文就是 flag 的 7 次方而没有得到 模 N 的约化。那么因为
$$
\begin{aligned}
ct&\equiv,m^e\mod,n,p^7|n\
\therefore ct&\equiv,m^e,mod,p^7\
\therefore m&\equiv,ct^{d},mod,p^7,d=e^{-1},mod,\phi(p^7)
\end{aligned}
$$

d = inverse(e, p**6*(p-1))
msg = pow(ct, d, p**7)
print(long_to_bytes(msg))
stage 2

这一关的p、q过于接近，可以尝试费马因子分解法 https://en.wikipedia.org/wiki/Fermat%27s_factorization_method。亦或者直接对模数开方在附近寻找素数。

cvctf{Hastad_with_e=65537_might_be_slow}

Big Rabin

题目描述：

Rabin cryptosystem, but big.

题目源码

from Crypto.Util.number import *
from secret import flag
import os, functools

primes = []
e = 2

for _ in range(10):
    primes.append(getPrime(1024))

n = functools.reduce((lambda x, y: x * y), primes)
m = bytes_to_long(os.urandom(256) + flag)
c = pow(m,e,n)

print(primes)
print(c)

分析

注意到 模数 n 是由 10个 256字节的数相乘得来，所以 n 总共有2560字节。所以即使 m 是在 flag 的基础上额外填充了 256 字节，但是 过于小的 e=2 使得 m 的 e 次方在模 n 下可能未得到约化。所以我们可以直接对 c 开 e 次方根恢复 m 。

cvctf{r4b1n_Cryp70_K1nd4_c0mpL1C4t3d?}
