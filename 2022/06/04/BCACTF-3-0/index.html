Hint 2 of 2

Each pair of letters decodes the next.

题目提示说使用了仿射密码。而且告诉我们前一个字母对可以解密下一个字母对，这说明前一个字母对应该就是仿射密码中的a、b值。编写脚本求解即可。


from Crypto.Util.number import inverse
from string import digits,ascii_lowercase
charset = digits+ascii_lowercase+"_"
ord = lambda c: charset.index(c)
current_key = "bc"
for current_char_c in "bx 6e z_ un uf i3 bm 0r 0x eb".split(" "):
    print(current_key, end="")
    a = ord(current_key[0])
    b = ord(current_key[1])
    m = len(charset)
    c = inverse(a, m)
    decrypt = lambda ct: charset[(c * (ord(ct)-b))%m]
    current_key = decrypt(current_char_c[0])+decrypt(current_char_c[1])

print(current_key)

fn get_line_stdin(stdin: &Stdin) -> String {
    let mut string = String::new();
    stdin.read_line(&mut string).unwrap();

    let mut string_characters = string.chars();
    string_characters.next_back();
    string_characters.as_str().to_owned()
}

pub fn decode_hex(s: &str) -> Result<Vec<u8>, ParseIntError> {
    (0..(s.len() / 2) * 2)
        .step_by(2)
        .map(|i| u8::from_str_radix(&s[i..i + 2], 16))
        .collect()
}


/// I didn't want to use a for loop. I am deeply sorry.
pub fn base64_decode(input: &str) -> Vec<u8> {
    const BASE_64_INDEX_MAP: &str = "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/";

    input.chars().skip(0).step_by(4).zip(
        input.chars().skip(1).step_by(4).zip(
            input.chars().skip(2).step_by(4).zip(
                input.chars().skip(3).step_by(4)
            )
        )
    )
        .map(
            |(a, (b, (c, d)))| {
                let mut bit_count = 0;
                (
                    [a, b, c, d]
                        .iter()
                        .map(|chr| 
                            BASE_64_INDEX_MAP.find(*chr).map(|i| usize::to_le_bytes(i)[0])
                        )
                        .inspect(|v| if v.is_some() { bit_count += 6 })
                        .map(|optional_value| optional_value.unwrap_or(0))
                        .fold(0u32, |num, n| (num << 6) + u32::from(n)),
                    bit_count
                )
            }
        )
        .map(|(int, bit_count)| (u32::to_be_bytes(int), bit_count))
        .map(|(bytes, bit_count)| bytes.iter().take(bit_count).cloned().collect::<Vec<_>>())
        .flatten()
        .collect()
}

fn sanitize_to_string<'a>(input: impl IntoIterator<Item = &'a u8>) -> String {
    String::from_utf8_lossy(&input.into_iter().cloned().filter(|byte| !char::is_ascii_control(&(*byte as char))).collect::<Vec<_>>()).to_string()
}

寻找到了一个使用DFS逐步搜索出密钥的答案，核心思想其实还是爆破密钥，但是加了一些优化操作使得算法实际运行时间比较快。

观察加密代码的核心就会发现加密算法将输入的明文填充到分组长度的整数倍，然后将填充之后的明文分组分成N组，随后用同一个密钥对这N组明文分组块进行加密（ECB模式）。注意到一个分组的长度是24bytes。

对一个分组块加密的具体操作如下：

首先密钥是一个24*24的由0、1构成的比特串，同时加密算法的输出是一个长为24的密文bytes。

加密规则：如果第i行第j列的key的值为1，那么就将当前的密文byte enc[i]异或上明文分组中的某个byte ptxt[j]，所以一个密文byte实际上是一个明文分组中某些bytes的异或。

解题脚本采用dfs逐步搜索的方式找到这个密钥key即flag。

首先第一步利用我们得到的一对明密文对，将其转化为加密算法的输入与输出。

第二步枚举所有可能的密钥分组，一个密钥分组长为24bit也就是三个bytes，由于密钥分组是由flag转变而来，所以密钥分组肯定是处于可以打印的字符范围内的bytes，这就明确了枚举密钥分组的边界在什么地方。

核心的问题来了：如何去判断枚举的密钥分组是正确的呢？这也是我们解题的关键一步也是最后一步。

答：编写dfs脚本搜索密钥。抽取出密文的某一列全部元素，比如抽取第1列的全部元素，那么这些元素代表的就是enc[0],enc[0+24],........enc[20+24]（分组之后明文只有20块，所以只能到20）

根据前面的分析这些元素是由同一个密钥分组加密而来。比如enc[0]所用密钥分组跟enc[0+24]所用密钥分组是一致的，这也就是说我们可以一块块地遍历明文分组，然后用我们枚举的密钥分组块充当前面加密算法的密钥，随后模仿加密算法的过程进行加密，之后查看当前位置的加密结果跟enc[i]是否相同。如果相同则说明枚举的密钥是对的，我们继续抽出下一列。

抽象地说其实就是固定住枚举的密钥分组的某一个分组块，用这个分组块对多个明文分组加密（写到这里感觉特别像维吉尼亚密钥体制了），如果枚举正确，那么加密得到的结果应该跟enc[i] enc[i+24] ….拼接起来是一样的
