文件上传漏洞总结

文件上传漏洞总结

文件上传

“文件上传”功能已经成为现在Web应用的一种常见需求，它不但有助于提供业务效率(例如：企业内部文件共享)，更有助于优化用户的体验（例如：上传视频、图片、头像等各种其他类型的文件）。”文件上传”功能一方面带来了良好的体验，另一方面也带来了“安全问题”。目前文件上传漏洞已经成为web安全中经常利用到的一种漏洞形式，对于缺少安全防护的web应用，攻击者可以利用提供的文件上传功能将恶意代码植入到服务器中，之后再通过url去访问以执行代码达到攻击的目的。

文件上传漏洞的成因

造成文件上传漏洞的原因有：

  • 开源编辑器的上传漏洞
  • 服务器配置不当
  • 本地文件上传限制被绕过
  • 过滤不严或被绕过
  • 文件解析漏洞导致文件执行
  • 文件路径截断

文件上传漏洞需要满足的条件（同时满足）

  • 恶意文件可以成功上传
  • 知道恶意文件上传后的路径
  • 恶意文件可以被访问或执行

文件上传过程中的检测

文件上传漏洞危害极大，所以我们必须对用户上传的文件做好检测，防止恶意用户对我们的服务器进行攻击。那为了防范文件上传漏洞，我们都可以做那些检测呢？

从上图可以看到，服务器端的检测主要就是查后缀、查内容。

一般一个文件上传过程中的检测分为如下几个部分：

  • 客户端 javascript 检测（通常为检测文件扩展名）
  • 服务器 MIME 类型检测（检测 Content-Type 内容）
  • 服务端目录路径检测（检测跟 path 参数相关的内容）
  • 服务端文件扩展名检测（检测跟文件 extension 相关的内容）
  • 服务端文件内容检测（检测内容是否合法或含有恶意代码）

检测绕过

如何分辨文件上传是客户端检测还是服务器端检测？

在常规的渗透测试过程中，我们根本无法获取到目标网站对于上传文件校验的源代码设计，那么如何判断文件是客户端校验还是服务器端校验呢？我们可以通过 burpsuite 来进行简易判断，具体流程如下：

  1. 首先打开 burpsuite 的代理模式，捕捉本机所有 http 请求。
  2. 在本地创建一个 shell.php 文件或是其他的木马文件并在网页上传（注意这里必须要是非法的后缀名，否则就通过上传了，这样我们就会捕捉到数据包）。
  3. 如果 burpsuite 未捕获到通信数据包且浏览器端已完成对该文件的校验，那么这种方式就属于客户端校验。

当然，真实环境中肯定是前端和后端同时进行检测，毕竟前端检测根本不安全。

通过禁用前端 JS 绕过前端 JavaScript 检测

以火狐浏览器为例说说如何关闭浏览器的 JS 功能：

  • 在 Firefox 地址栏里输入 about:config
  • 在搜索栏输入 javascript.enabled 查找到首选项

通过 burpsuite 抓包并修改数据包的方法绕过前端 JS 检测

利用 burpsuite 可以轻松绕过前端检测，具体的做法如下：

  1. 打开 burpsuite 代理模式，捕获本机的 http 请求。
  2. 将要上传的恶意文件后缀名改为合法的后缀名并上传。（这样我们就可以通过前端的校验）
  3. burpsuite 会捕获到上传的 http 请求包，此时我们在请求包中将我们的文件后缀名改回原本的形式。

文件后缀名绕过

为了能在服务器上执行代码，我们上传的木马文件肯定不可以是图片的后缀名，如果此时服务器端采用的是白名单检测后缀名，我们如何在保持木马文件后缀名的情况下成功上传呢？

00截断绕过白名单后缀名检测（针对PHP）

00截断是很常见的截断方法，原理是有些函数处理时会将00这个空字符当作结束符，从而提前结束对相关数据的处理。

截断条件：php 版本小于 5.3.4，php 的 magic_quotes_gpc 为 OFF 状态。

示例：test.php%00.jpg,test.php0xoo.jpg

值得注意的是，在 GET 型 00 截断中，我们要使用 %00 来截断，因为 GET 型提交的内容会被自动进行 URL 解码；而在 POST 型 00 截断中，%00 是不会被自动解码的，我们需要在 16 进制中对其进行修改，将其改为 00 即可截断。
