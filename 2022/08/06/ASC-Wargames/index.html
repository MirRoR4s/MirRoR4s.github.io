题目：

这个脚本有两个函数，分别是gen 和 encrypt。gen 函数生成加密所用的公钥以及一个私密值（应该是解密用的）。encrypt 则是用gen 函数生成的公钥对flag进行某种加密。

gen 函数首先生成一个256比特的素数，其中的低25位比特是随机的；即$p = 2^{255} + 2^{127} + rand(2^{25})$。

随后函数进入循环结构中，进行25轮循环迭代。每一轮循环生成两个随机数 $q$, $r$ ,之后用如下等式生成这一轮的公钥值 $ pub_i = pq_i + 2r_i$

encrypt 函数将公钥和flag作为输入，随后枚举flag中的每一个比特，对其作如下的操作：

首先从公钥中得到5个随机数作为$ids$ ，之后再生成一个随机的128比特的 $r$，然后用如下的等式加密当前遍历的flag的比特值：

$C_i = 2r + Flag_{bit} + \sum ids$

求解：

因为gen 函数生成的素数 $p$ 的只有最低位的25个比特不知道，所以我们暴力破解得到 $p$。但是在找到p之后，我们的下一步该如何去做？或者说我们如何不知道$2r+\sum ids = 2r+\sum (pq_i +2r_i)$ 的情况得到 $Flag_{bit}$

利用模运算的性质，如果我们将上述等式模上p，那么就会得到$2r+\sum 2r_i+Flag_{bit}\equiv C_i \mod p$

如果我们再将上述等式模上2，那么就会得到$Flag_{bit} \equiv C_i \mod 2$，$C_i$的值我们是知道的，所以最终我们可以解出$Flag_{bit}$。

Vault

Apparently, the order is factored int small primes and one of the them is exactly the order of the generator point, so we can assume that the given point is generated by dividing the curve order by 12708491 then multiplied by the original generator G = (o//12708491)*E.gens()[0] that will produce a point with order or 12708491.

Notice that, any generated points from G will have the same order of 12708491, so we can brute force the point from the range [1,12708491][1,12708491], try to decrypt the message, and check if it contains the flag or not.
