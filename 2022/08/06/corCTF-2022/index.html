一开始我的想法是找到某个x的值，使得LCG的值不会随着迭代改变。但是显然我找不到这样的，所以我没有解出这道题。事实证明我想的太复杂的，只需要进行简单的同余运算，这道题就可以求解出来。

$x\equiv ax + b \mod p$

$(1-a)x \equiv b \mod p$

因为 $p$ 是素数，所以$(1-a)$ 在模 $p$ 下的逆元大概率存在，所以我们可以得到

$x\equiv b\ (1-a)^{-1} \mod p$

而 $a$ 、 $b$ 是固定的值，所以 $x$ 就这么求出来了。

阅读完源代码发现有点类似于Diffle-Hellman密钥交换，不同的是现在要对s进行a_priv次 f(s) 操作。从数学的角度分析一下：

$S_0 = S$

$S_1 = aS_0+b \mod p $

$S_2 = aS_1 +b \mod p$

以此类推得到

$S_i = aS_{i-1}+b \mod p $

令 a_priv = $ka$ , b_priv = $kb$

所以 A = mult(s, a_priv) 等价于 $S_{ka}\equiv aS_{ka-1}+b \mod p$

B = mult(s, b_priv) 等价于 $S_{kb}\equiv aS_{kb-1}+b \mod p $

那么mult(A, b_priv) 就等价于 $S_{ka+1},S_{ka+2},…,S_{ka+kb}$ = mult(B,a_priv)

因此二者是相等的，所以可以成功共享密钥。

解法1：离散对数

https://connor-mccartney.github.io/cryptography/other/corCTF-2022

$f(s)=as+b\mod p \f(f(s))=a(as+b)+b=a^2s+ab+b\mod p\f(f(f(s)))=a(a^2s+ab+b)+b=a^3s+a^2b+ab+b \mod p$

注意到上述公式除了最高次项之外，其余项可以写为和的形式，即 $f^n(s)=a^ns+b\sum_{i=1}^na^{i-1}=a^ns+ \frac{b(1-a^n)}{1-a}\mod p$

令 $u=(1-a)^{-1}\mod p $

则 $\begin{eqnarray} A &\equiv& a^ns+b(1-a^n)u \mod p \nonumber \ A&\equiv& a^ns+bu-ba^nu \mod p \nonumber \A-bu&\equiv& a^n(s-bu) \nonumber \(A-bu)(s-bu)^{-1}&\equiv& a^n \mod p \nonumber \end{eqnarray} $

上述等式左边是已知的值，模数 $p$ 也知道。所以现在的问题就转化为了离散对数问题，幸运地是，由于 $p-1$ 是光滑的，所以我们可以利用pohlig-hellman解出n。

#!/usr/local/bin/python
import random
import time
import math
import binascii
from Crypto.Util.number import *

p, q = getPrime(512), getPrime(512)
n = p * q
phi = (p - 1) * (q - 1)

flag = open('./flag.txt').read().encode()

random.seed(int(time.time()))

def encrypt(msg):
    e = random.randint(1, n)
    while math.gcd(e, phi) != 1:
        e = random.randint(1, n)
    pt = bytes_to_long(msg)
    ct = pow(pt, e, n)
    return binascii.hexlify(long_to_bytes(ct)).decode()

#!/usr/local/bin/python
from Crypto.Util.number import getPrime, inverse, bytes_to_long
from random import randrange

with open("flag.txt", "rb") as f:
	flag = f.read().strip()

def gen_keypair():
	p, q = getPrime(512), getPrime(512)
	n = (p**2) * q
	while True:
		g = randrange(2, n)
		if pow(g, p-1, p**2) != 1:
			break
	h = pow(g, n, n)
	return (n, g, h), (g, p, q)

def encrypt(pubkey, m):
	n, g, h = pubkey
	r = randrange(1, n)
	c = pow(g, m, n) * pow(h, r, n) % n
	return c

def decrypt(privkey, c):
	g, p, q = privkey
	a = (pow(c, p-1, p**2) - 1) // p
	b = (pow(g, p-1, p**2) - 1) // p
	m = a * inverse(b, p) % p
	return m

def oracle(privkey, c):
	m = decrypt(privkey, c)

解题方法一：二分查找迫近p

本题实现了 Okamoto-Uchiyama cryptosystem ，这也是一个公钥密码体制。现在我们拥有加密后的flag，并且我们可以解密任意密文并得到解密结果的最低位，很自然的就可以想到RSA的LSB oracle，使用类似的思想，看看能否从这个oracle中恢复出私钥的信息。

注意到解密的信息会进行模 $p$ 的操作（在本题的密码体制中，公开模数 $n=p^2q$ ， $p$ 和 $q$ 组成了私钥。），这对我们很有用，因为我们可以从已知明文的解密结果收集到 $p$ 的信息。

我们用 $\mathcal{O}$ 表示本题的oracle，于是我们有：

$\left{ \begin{array}{rcl}1,& if\ decrypt(c)\ is\ odd \ 0,& otherwise\end{array}\right.$

假设我们有一个小于p的信息m，考虑一下我们能从$\mathcal{O}(encrypt(2m))$ 中获得关于p的什么信息。

如果该oracle返回1，那么我们一定有 $p<2m$ 。你可能会疑惑难道 $p<2m$ 就不会返回0么？

让我们来分析一下，当 $p<2m$ 时，根据带余除法我们有：

$2m=k*p+r,\ 0<=r<p$

因为 $2m$ 是偶数，$kp$ 是奇数，所以 $r$ 必定是奇数，所以此时返回的结果一定是 $1$

当 $p>=2m$ 时，此时模 $p$ 什么都没做，直接返回 $0$ 。

因此根据以上两种状态，我们可以写一个二分查找算法来找到 $p$ 可能在的区间，从而拿到私钥。

因为 $p$ 是一个 $512$ 比特的素数，所以最多进行 $512$ 次二分查找就可以确定 $p$ 所在的区间。

from pwn import *

	b = (pow(g, p-1, p**2) - 1) // p
	m = a * inverse(b, p) % p
